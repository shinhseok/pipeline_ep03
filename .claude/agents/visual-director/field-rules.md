# Image Prompt v3 Field Rules — Visual Director Reference (순수 한국어 자연어)

> v3: 구조적 태그 제거. creative_intent → 서술 단락 직접 매핑.
> ref_images YAML 필드 + 서수 참조 패턴 사용.

---

## creative_intent → 서술 단락 매핑

| creative_intent tag | 서술 단락 | 변환 지침 |
|---------------------|----------|-----------|
| `[공간]` | P2 배경 + P4 환경 | 밀도 등급(L0~L5) → `v3_background_density` 서술 선택. L3+ → P4에 구조물 서술 |
| `[카메라]` | P3 크기 3중 + 위치 | 아키타입 + 앵글 → 프레임 점유율(%) + 위치 + 은유 |
| `[소품]` | P4 | 소품 형태, 크기, 공간 관계. prop_ref 있으면 서수 참조 |
| `[조명]` | P4 또는 P5 | 그림자/광원이 서사 핵심이면 P4에 묘사. 아니면 P5에 분위기 보강 |
| `[감정선]` | P3 표정/포즈 | emotion_tag에 맞는 구체적 표정·제스처를 P3에 자연 통합 |
| `[이전샷]` | P4 뒤 또는 P3 앞 | 이전 Shot과의 시각적 대비/연속성을 구도 근거로 활용 |

## ref_images 순서 규칙

**YAML `ref_images` 배열의 순서 = image_prompt 내 서수**

| 순서 | 대상 | THIS {name} 참조 |
|------|------|-----------------|
| 1번 | costume_ref (주인공) | `THIS artisan` |
| 2번 | prop_ref 또는 costume_ref_B | `THIS spinning_wheel` |
| 3번 | 추가 prop_ref | `THIS compass` |
| 4번 | (최대) | `THIS {name}` |

**배치 우선순위**: costume_refs > main_turnaround > prop_refs > character_prop

**경로**: `09_assets/reference/` 기준 상대 경로 (접두사 금지)
- `characters/run001/artisan.jpeg`
- `props/run001/compass.jpeg`
- `ai_robot.jpeg` (reference/ 직접)

**has_human 기반 ref_images 구성 규칙 (visual-director 완전 책임)**:

| has_human | costume_refs | ref_images에 포함할 항목 |
|-----------|-------------|-------------------------|
| main | `[]` (기본 해빛) | style_ref + main_turnaround + prop_refs |
| main | `[변장명]` | style_ref + characters/{RUN_ID}/{변장명}.jpeg + prop_refs |
| anonym | `[]` | style_ref + character_reference + prop_refs |
| none | - | style_ref + prop_refs (캐릭터 ref 없음) |

⚠️ **style_ref**는 모든 shot의 ref_images 첫 항목으로 포함 필수.
⚠️ **has_human: main이면 ref_images에 캐릭터 ref 1개 이상** 필수.
⚠️ **generate_images.py는 ref_images를 그대로 전달** — 추가/제거 없음.

## [공간] → 배경 밀도 등급 분기 테이블

[공간] 태그에 `L{N}` 등급이 있으면 해당 등급의 서술을 사용한다.
등급이 없으면 키워드 기반 자동 매핑 (하위 호환).

### 등급별 처리

| 등급 | 잉크 밀도 | P2 배경 서술 | P4 환경 서술 | 잉크 무게 지시 |
|------|-----------|-------------|-------------|--------------|
| **L0** (bare) | 0% | `v3_background_density.L0.prompt` | — | — |
| **L1** (trace) | ~2% | `v3_background_density.L1.prompt` (힌트 채움) | — | — |
| **L2** (whisper) | ~5% | `v3_background_density.L2.prompt` (편린 채움) | — | "배경 선은 캐릭터 선보다 확실히 연하고 가늘게" |
| **L3** (sketch) | ~10% | `v3_background_bare` 기조 유지 | P4에 구조물 서술 추가 (has_human:none 확장) | "배경 선은 캐릭터 선보다 확실히 연하고 가늘게" |
| **L4** (scene) | ~15% | `v3_background_bare` 기조 유지 | P4에 다중 구조물 서술 + 원근 선 굵기 차이 | "가까운 선 약간 굵게, 먼 선 더 가늘고 흐리게" |
| **L5** (panorama) | ~18% | `v3_background_bare` 기조 유지 | P4에 풍경 서술 + 3단계 깊이감 | "원경 극도로 가늘게, 중경 약간 뚜렷, 근경 비교적 또렷 — 모두 캐릭터보다 연하게" |

### 등급 미기재 시 키워드 자동 매핑

| [공간] 키워드 | 자동 매핑 |
|--------------|----------|
| "빈 캔버스", 환경 미언급, AWE Shot | L0 |
| "수평선", "바닥선", "그림자" | L1 |
| 환경 편린 2~3개 | L2 |
| 단일 구조물 ("문틀", "나무") | L3 |
| 다중 구조물 ("방+가구") | L4 |
| 풍경 ("지평선+건물군") | L5 |

### 핵심 규칙

- **배경 요소에 채색 절대 금지** — 잉크 라인만
- **L2 이상**: "배경 선은 캐릭터 선보다 확실히 연하고 가늘게" 필수
- **L4~L5**: 원경/근경 선 굵기 차이로 깊이 표현
- **잉크 무게 계층**: 캐릭터(1.0x) > 소품(0.7x) > 배경(등급별 0.25~0.7x)

### 텍스트 렌더링 금지 규칙

NB2는 프롬프트에 텍스트 묘사가 있으면 이미지에 글씨를 렌더링하려 시도한다. 대부분 깨지거나 스타일을 망치므로 원칙적으로 금지.

| 허용 | 금지 |
|------|------|
| 숫자 + 단위 ("30", "6", "144", "4") | 한글 문장/단어 |
| 시간 표현 ("30시간", "6시간", "하루", "10분") | 영문 문장/단어 |
| 간단한 수식 기호 ("→", "×") | 설명 텍스트, 라벨, 캡션 |

image_prompt에서 텍스트 요소를 묘사할 때: "숫자 '30'과 '6'이 화면에 보인다" (O) / "런던-맨체스터라는 글씨가 보인다" (X)

> ⚠️ Text-Motion scene_type인 경우에도 이 규칙 적용 — 숫자/단위만 허용.

### 배경 묘사 핵심 원칙

> ⚠️ 배경이 복잡할수록 NB2가 Sempé 스타일을 이탈할 위험이 높다. 아래 원칙을 반드시 지킨다:
> 1. 배경은 **윤곽선/구조만** — 해칭, 그라데이션, 면 채우기로 톤을 만들지 않는다
> 2. 배경 선은 **캐릭터/소품 선보다 반드시 연하고 가늘게** — image_prompt P2에 매번 명시
> 3. 빈 공간은 **반드시 순백** — "분위기", "어둠", "톤"을 배경 면으로 표현하지 않는다

### L3+ 배경 P2 금지 표현 (NB2 톤 오염 방지)

NB2는 아래 표현을 **배경 전체의 톤/색을 어둡게** 만드는 것으로 해석한다:

| 금지 표현 | NB2 해석 | 대체 표현 |
|----------|---------|----------|
| "교차 해칭으로 어두움 암시" | 배경 전체를 회색으로 채움 | "윤곽선으로 구조만 암시" |
| "좁고 어두운 공장" | 배경에 짙은 톤 적용 | "공장 벽 윤곽이 가느다란 잉크 선으로" |
| "어두운 내부" / "그늘진" | 면 채우기로 어두움 표현 | "내부는 비워둬" |
| "깊이감을 표현" (L3에서) | 원근법 구조물 상세 묘사 | L3에서는 단일 구조물 윤곽만 |
| "빽빽하게" / "좁은 통로" | 공간을 채워 여백 제거 | "윤곽선만으로 구조를 암시" |
| **"~로 깔려" / "드리워져" / "퍼져" / "가득"** | **빈 공간을 워시/해칭으로 채움** | **구조물 윤곽만 그리고 "빈 공간은 순백"** |
| **"그림자로 깔려" / "분위기 조성"** | **배경 전체에 톤 확산** | **"구조물 아래에 해칭 두세 줄만"** |

**L3+ 필수 가드**: 모든 L3+ 배경 서술 말미에 **"구조물 외의 빈 공간은 반드시 순백으로 남겨줘"** 포함 필수.

### 흐림/연함 표현 금지 (NB2 이미지 품질 오염 방지)

NB2는 아래 표현을 **이미지 전체를 흐리게/연하게** 렌더링하는 것으로 해석한다:

| 금지 표현 | NB2 해석 | 대체 표현 |
|----------|---------|----------|
| "겨우 보일 정도" / "너무 연해서" | 이미지 전체 투명도 하락 | "가느다란 잉크 선으로" |
| "흐릿하게" / "바래져가는" | 이미지 전체 블러 | "잉크 밀도가 감소하는" |
| "안개에 삼켜진 듯" | 전체 화면 안개 효과 | "윤곽선이 끊기는" |
| "희미한 그림자" / "아련한" | 전체 톤 다운 | "가느다란 해칭 두세 줄" |

> ⚠️ 특정 요소만 연하게 하고 싶으면 "이 요소의 잉크 선만 가늘고 연하게" 식으로 대상을 한정한다. "전체적으로 연한" 식의 전역 지시 금지.

**L3 P2 올바른 패턴**:
```
배경은 순백의 빈 공간이되, {구조물} 윤곽이 얇고 연한 잉크 선으로만
암시되어 있어 — 내부는 비워둬. 구조물 외의 빈 공간은 반드시 순백으로 남겨줘.
배경 선은 캐릭터 선보다 확실히 연하고 가늘게.
```

## 크기 표현 3중 체계

캐릭터와 소품 모두 반드시 3가지 방식으로 크기를 표현한다:

| 방식 | 예시 | 목적 |
|------|------|------|
| **수치** | "전체 화면의 약 10%" | 정밀 제어 |
| **상대비교** | "물레의 절반 크기로" | 비례감 전달 |
| **은유** | "마치 거대한 세계 속 하나의 점처럼" | 감정적 스케일 |

> ⚠️ **소형 캐릭터 비율 유지 (≤10%)**: 캐릭터가 화면의 10% 이하로 작게 그려질 때, "THIS main의 체형 비율(둥근 머리, 짧은 팔다리, 강낭콩 실루엣)을 정확히 유지해줘"를 P3에 반드시 포함. 작은 크기에서 턴어라운드 시트의 디테일이 유실되는 것을 방지.

## 감정 표현 → P3 자연 통합 (뉘앙스 인식)

emotion_nuance가 있으면 `sempe-ink.yaml` E11 (`v3_emotion_direction`)에서
해당 뉘앙스의 `face` + `upper_body` + `gesture`를 **3파트로 통합**하여 P3에 녹인다.

emotion_nuance가 없으면 기존 `v3_emotion_expressions` 폴백.

### 뉘앙스 → P3 3파트 통합 패턴

```
{face 묘사}, {upper_body 묘사} 채 {gesture 묘사}하고 있어.
```

**예시:**
- **slapstick**: "입이 오벌로 벌어진 채 상체가 스프링처럼 튀어오르며 양팔을 벌려 허우적대고 있어."
- **contemplative**: "시선을 아래로 떨구고 입을 조용히 다문, S자 곡선으로 서서 멀리 놓인 소품을 조용히 바라보고 있어."
- **pressure**: "이마에 주름을 잡은 채 몸통이 찌그러진 듯 아래로 압축된, 소품이 위에서 내려누르는 모습이야."

### 뉘앙스 → P2·P4·P5 응집

뉘앙스의 7차원 중 나머지 4차원은 P3 밖에서 작용한다:

| 차원 | 반영 단락 | 작용 방식 |
|------|----------|----------|
| `bg_density` | P2 배경 | 밀도 등급(L0~L5) → 해당 서술 선택 |
| `prop_relation` | P4 소품 | 크기 대조 + 공간 관계 서술에 반영 |
| `camera` | P3 크기 3중 | 앵글·거리·점유율에 반영 |
| `color_point` | P5 색 계획 | 채색 대상·색상 선택에 반영 |

### 7차원 톤 일관성 교차 검증

image_prompt 작성 완료 후, P2~P5가 동일 뉘앙스의 톤을 공유하는지 확인:
- P3의 표정/포즈 ↔ P2의 배경 밀도: 긴장된 포즈인데 L0 순백 → 톤 불일치 가능
- P4의 소품 크기 대조 ↔ P3의 캐릭터 반응: 압도 소품인데 태연한 포즈 → 불일치
- P5의 색 ↔ P3의 감정: 압박 분위기인데 warm gold → 불일치

### 레거시 폴백 (emotion_nuance 없을 때)

| emotion_tag | P3 삽입 표현 |
|-------------|-------------|
| `REVEAL` | "눈이 크게 떠지고 입이 벌어진 채 뒤로 살짝 물러서는" |
| `REFLECTIVE` | "시선을 아래로 떨구고 입을 조용히 다문 채 가만히 있는" |
| `HUMOR` | "한쪽 눈썹을 치켜올리고 입꼬리가 비스듬히 올라간" |
| `AWE` | "눈이 동그랗게 커지고 입이 살짝 벌어진 채 멈춰 있는" |
| `TENSION` | "눈을 가늘게 좁히고 이마에 주름을 잡은 채 어깨를 잔뜩 움츠린" |

## 채색 원칙

### 캐릭터 색 = 턴어라운드 시트 위임

has_human: main/anonym일 때, **캐릭터 색은 ref 이미지(턴어라운드 시트)가 담당**.
image_prompt P4에서 캐릭터 색을 별도 지정하지 않는다.
스타일·얼굴을 ref에 위임한 것과 동일한 원칙.

P4에 **"캐릭터는 참조 이미지의 색을 그대로 유지해줘"** 문구를 반드시 포함.

### 장면 포인트 색 — 2중 배치 (NB2 색 누락 방지)

NB2는 프롬프트 마지막 문장의 우선순위를 낮게 처리한다.
**장면 포인트 색**은 P3 인라인 + P4 강조 2곳에 배치.

**P3 인라인** — 소품/환경 묘사에 색을 통합:
```
화면 우측에는 THIS spinning_wheel이 따뜻한 amber 빛을 머금고...
```

**P4 강조** — "반드시" + 유일한 색 강조:
```
반드시 THIS spinning_wheel에만 deep amber 워시를 입혀줘 — 캐릭터는 참조 이미지의 색을 그대로 유지해줘.
```

### 금지 표현

| 금지 | NB2 해석 | 대체 |
|------|---------|------|
| "살짝 입혀줘" | 안 해도 됨 | "반드시 입혀줘" |
| "워시 채색을 살짝" | 극소량 → 안 보임 | "워시를 입혀줘" |
| P4에만 채색 지시 | 우선순위 낮음 → 무시 | P3 인라인 + P4 강조 |
| 캐릭터 색 직접 지정 | ref 시트와 충돌 가능 | "참조 이미지의 색 그대로" |

## creative_intent 변환 원칙 (개념 → NB2 실행 가능 지시)

shot-composer의 creative_intent는 **개념적 표현**이다.
visual-director는 이를 **NB2가 리터럴하게 실행해도 올바른 결과가 나오는 구체적 시각 지시**로 변환해야 한다.

### 변환 규칙

| creative_intent 개념 | 잘못된 리터럴 전사 | 올바른 NB2 실행 가능 지시 |
|---------------------|-------------------|------------------------|
| "검은 blob으로 개별성 상실" | "짙은 잉크 덩어리 같은 실루엣" | "어두운 잉크로 채워진 콩 캐릭터 실루엣 — 둥근 머리와 짧은 팔다리는 유지하되 얼굴과 의상은 생략" |
| "압도하는 어두움" | "교차 해칭으로 어두움 암시" | "구조물 윤곽이 가느다란 잉크 선으로 — 내부는 비워둬" |
| "갇힌 공간" | "좁고 어두운 내부" | "벽 윤곽이 얇은 잉크 선으로 암시 — 내부는 순백" |
| "납작한 형체" | "의상이나 얼굴 없는 형체" | "콩 캐릭터 기본 형태의 어두운 실루엣" |

### 실루엣 군중 표현 규칙

실루엣이라도 **콩 캐릭터의 기본 형태**를 반드시 유지:
- **유지**: 둥근 머리, 통통한 몸통, 짧은 팔다리
- **제거**: 얼굴, 의상 디테일, 개별 표정
- **금지 표현**: "잉크 덩어리", "납작한 형체", "blob", "형체 없는"

올바른 군중 묘사 (솔리드 블랙 금지 — 잉크 윤곽선+해칭으로 표현):
```
뒤쪽에는 성긴 해칭으로 어둡게 표현한 콩 캐릭터 실루엣 {N}개가 서 있어 —
둥근 머리와 짧은 팔다리는 보이지만 얼굴이나 의상 디테일은 없어.
앞쪽 {M}개만 faded grey wash로 약간 밝고, 나머지는 빽빽한 해칭 실루엣이야.
```

크기 대비가 있는 군중 묘사 (어른 vs 아이 등):
```
왼쪽에 키가 큰 콩 캐릭터 {M}명(약 12%)이 잉크 윤곽선으로 서 있고,
그 옆으로 키가 절반인 작은 콩 캐릭터 {N}명(약 6%)이 빽빽하게 서 있어.
모두 떨림 있는 잉크 윤곽선으로 그리되, 솔리드 블랙 채움은 쓰지 마.
```

### main vs anonym 판단 기준

- **main**: 그 캐릭터가 **누구인지 알 수 있다** (의상·맥락·이름으로 특정 가능)
- **anonym**: 사람 형태는 있지만 **누구인지 알 수 없다** (익명·상징적·군중)

## ref_images 캐릭터 소스 규칙

> ⚠️ **경로 단일 소스**: 모든 ref_images 경로는 **ANCHOR.md의 `[REF PATH REGISTRY]`**에서 참조한다. 아래 표의 경로는 예시이며, 실제 작업 시 반드시 ANCHOR를 읽어 정확한 경로를 확인한다.

| has_human | 조건 | ref_images 포함 |
|-----------|------|----------------|
| `main` | costume_refs 있음 | costume_ref 턴어라운드 시트 (ANCHOR ref_paths 참조) |
| `main` | costume_refs 없음 | ANCHOR ref_paths.main_turnaround |
| `main` | secondary_chars 직접 등장 | 캐릭터별 턴어라운드 각각 포함 + THIS {name} 구분 |
| `main` | secondary_chars: [happy_rabbit] | style_ref + main_turnaround + happy_rabbit.jpeg |
| `anonym` | — | ANCHOR ref_paths.character_reference |
| `none` | — | 캐릭터 ref 없음 |

### 신체 표현 규칙

캐릭터의 **신체 일부**(손, 팔, 실루엣, 그림자 등)만 등장할 때:

**main인 경우** (특정 캐릭터의 손/실루엣):
1. ref_images에 해당 캐릭터 턴어라운드 **반드시 포함**
2. image_prompt에서 **`THIS {character_name}의 손/실루엣`** 패턴 사용

```
❌ "콩 캐릭터 손 한 쌍" — 제네릭 표현
✅ "THIS main의 손 한 쌍" — 명시적 캐릭터 참조
```

**anonym인 경우** (익명의 손/실루엣):
1. ref_images에 `assets/reference/style/character_reference.jpeg` 포함
2. image_prompt에서 **`THIS character_reference의 형태를 따른 실루엣/손`** 패턴 사용

```
❌ "콩 캐릭터 실루엣 3개" — ref 없는 제네릭 표현
✅ "THIS character_reference의 형태를 따른 실루엣 3개" — 기본 형태 참조
```

> visual-director가 ref_images를 완전히 구성한다. generate_images.py는 추가/제거 없이 그대로 전달.
