# Content Planner — Section Guide (15개 섹션 상세 작성 규칙)

---

## 섹션 1~10 상세 규칙

### §1 핵심 메시지 설계

**3층 메시지 구조:**

| 층 | 이름 | 설명 | 형식 |
|----|------|------|------|
| **Layer 1** | 한 줄 메시지 (One-Liner) | 영상 전체가 전달하는 단 하나의 주장. 시청자가 친구에게 "이 영상은 ~에 대한 건데, 핵심은 ~이야"라고 말할 수 있는 문장 | 1문장 (30자 이내 권장) |
| **Layer 2** | 3개 전달 포인트 | 한 줄 메시지를 뒷받침하는 하위 주장 3개. 각 포인트는 내러티브 단계(셋업·텐션·페이오프)와 1:1 대응 | 각 1문장 |
| **Layer 3** | 증거·사례 | 각 전달 포인트를 뒷받침하는 구체적 데이터, 사례, 메타포 | 핵심 질문·보조 인사이트에서 도출 |

**"친구 테스트" ⚠️** — 기획 완성 후 아래 문장을 자연스럽게 채울 수 있는지 검증:

> "이 영상은 **{주제}**에 대한 건데, 핵심은 **{한 줄 메시지}**야.
> 왜냐면 **{전달 포인트 1}**이고, **{전달 포인트 2}**이거든.
> 그래서 **{전달 포인트 3}**를 제안하더라."

이 문장이 어색하면 한 줄 메시지가 불명확한 것 — 재설계 필수.

**메시지 반복 전략 (script-director에 전달):**

| 등장 횟수 | 형태 | 위치 |
|----------|------|------|
| 1회 | 직접 진술 — 명확한 문장으로 | 텐션 섹션 전환점 |
| 2회 | 메타포/비유 — 이미지로 | 셋업 또는 텐션 |
| 3회 | 질문형 재진술 — 시청자에게 되묻는 형태 | 아웃트로 |

> ⚠️ 3회 미만이면 시청자가 핵심 메시지를 놓칠 가능성이 높다. 3회 초과는 가능하나, 동일 표현 반복 금지 — 매번 다른 각도에서 재진술.

### §9 메시지 앵커링 맵

각 내러티브 단계에서 핵심 메시지가 어떤 형태로 표면화되는지 명시:

| 내러티브 단계 | 메시지 역할 | 앵커링 방식 |
|-------------|-----------|-----------|
| **훅 (Hook)** | 한 줄 메시지의 문제 면을 감각적으로 보여줌 | 시청자가 "뭔가 이상한데?"라고 느끼게 하는 장면·질문 |
| **셋업 (Setup)** | 전달 포인트 ①: 왜 이것이 중요한가 | 배경·맥락으로 "나도 해당되는 문제"임을 인식시킴 |
| **텐션 (Tension)** | 전달 포인트 ②: 문제의 메커니즘/본질 | 메타포를 통해 복잡한 구조를 직관적으로 "아하!" 전환 |
| **페이오프 (Payoff)** | 전달 포인트 ③: 그래서 어떻게 할 수 있는가 | 즉시 실행 가능한 방법론으로 해소감 제공 |
| **아웃트로 (Outro)** | 한 줄 메시지를 새로운 시각으로 재진술 | Hook 이미지 순환 + 오픈 루프로 여운과 다음 행동 유도 |

### §10 비주얼 스토리텔링 전략

핵심 메시지를 시각적으로 전달할 **비주얼 모티프(Visual Motif)**를 설계:

| 요소 | 설명 | 기획 시 기술 |
|------|------|------------|
| **비주얼 모티프** | 핵심 메시지를 상징하는 시각 요소 1~2개. 영상 전체에서 반복·변형되며 메시지를 강화 | 예: "잠긴 문 → 열린 문", "마른 나무 → 녹색 잎" |
| **모티프 변형 계획** | 모티프가 내러티브 단계별로 어떻게 변형되는지 설계 | 훅(닫힘) → 텐션(균열) → 페이오프(열림) → 아웃트로(새 문) |
| **키 비주얼 Shot** | 핵심 메시지가 가장 강렬하게 시각화되는 결정적 Shot 1~2개 | 텐션 전환점 + 아웃트로 순환 Shot |
| **씬 유형 사전 제안** | 각 Section에 적합한 씬 유형을 미리 제안 → shot-composer가 참고 | 아래 표 참조 |

**섹션별 씬 유형 사전 제안 (shot-composer 참고용):**

| Section | 권장 씬 유형 | 이유 |
|---------|------------|------|
| HOOK | `Doodle-Character` | 캐릭터 임팩트로 호기심 점화 |
| SECTION01 | `Doodle-Animation`, `Doodle-Illust` | 배경·맥락 설명에 적합 |
| SECTION02 | `Doodle-Illust`, `Whiteboard-Reveal` | 메타포 시각화, 개념 전개 |
| SECTION03 | `Doodle-Diagram`, `Text-Motion` | 실천 해법 구조화 |
| OUTRO | `Doodle-Character` | Hook과 시각적 대응·순환 |

> ⚠️ 비주얼 모티프는 shot-composer가 creative_intent 설계 시 관통 요소로 활용한다.
> 키 비주얼 Shot은 이미지 생성 시 최우선 품질 관리 대상이 된다.
> 씬 유형 제안은 shot-composer에 대한 참고이며, 최종 결정은 shot-composer가 한다.

### §11 Hook 제작 옵션

기획 단계에서 Hook의 **타입**과 **미디어 형식**을 선택한다. 두 옵션은 독립적(직교적)으로 선택 가능.

**Hook 타입 (HOOK_TYPE):**

| 값 | 설명 |
|----|------|
| `standard` (기본) | 기존 나레이션 Hook — 변경 없음 |
| `song` | 30초 싱어송라이터 노래 Hook — Shorts 듀얼 활용 |

Song Hook 선택 시 추가 기술:
- **가사 컨셉**: 에피소드 주제를 유머러스하게 요약하는 방향
- **톤**: 웃긴 싱어송라이터, 자기비하적, 기타 반주
- **시그니처 멜로디**: 채널 공통 멜로디(Suno 생성) — 가사만 에피소드별 변경
- **브릿지 설계**: 노래 → 나레이션 전환 방법 (후크라인 질문화 → 침묵 → 나레이션)
- **가사 분량**: 80~120 음절, 6~10줄 (중간 템포 ~4음절/초 × 30초)

**Hook 미디어 (HOOK_MEDIA):**

| 값 | 설명 |
|----|------|
| `image` (기본) | 기존 정적 이미지 + 나레이션 |
| `video` | Veo3/Kling 영상 클립 — Start/End 이미지 + 모션 프롬프트 |

Video Hook 선택 시 추가 기술:
- **영상 엔진**: `veo3` (기본) / `kling`
- **Shot별 이미지 장수**: 1장(Start only) vs 2장(Start + End) — Shot 성격에 따라 선택
- **이미지 생성 우선순위**: Hook 이미지가 영상 생성의 입력이므로 Phase 0으로 우선 생성

**4가지 조합:**

| HOOK_TYPE | HOOK_MEDIA | 결과 |
|-----------|-----------|------|
| standard | image | 기존 방식 (나레이션 + 이미지) |
| standard | video | 나레이션 + 영상 클립 |
| song | image | 노래 + 이미지 |
| song | video | 노래 + 영상 클립 (최대 임팩트) |

### §12 YouTube 메타데이터 설계

영상 업로드 시 필요한 메타데이터를 기획 단계에서 **초안** 작성한다.

**YouTube 업로드 필수 항목:**

| 항목 | 기획 시 결정 | 최종 확정 시점 |
|------|------------|-------------|
| **제목 (Title)** | 섹션 5에서 3개 후보 → 여기서 최종 1개 선택 | 업로드 전 |
| **설명 (Description)** | 구조 + 핵심 문장 초안 | 영상 완성 후 타임스탬프 추가 |
| **태그 (Tags)** | 메인 3개 + 롱테일 7~10개 | 업로드 시 |
| **해시태그 (Hashtags)** | 노출용 3개 + 설명란 5~10개 | 업로드 시 |
| **썸네일 전략** | 텍스트 + 비주얼 컨셉 + 감정 방향 | 영상 완성 후 |
| **카테고리** | `Education` 고정 | 고정 |
| **영상 언어** | `한국어` 고정 | 고정 |
| **아동용 여부 (COPPA)** | `아동용 아님` 고정 | 고정 |
| **라이선스** | `Standard YouTube License` 기본 | 고정 |
| **플레이리스트** | 소속 시리즈/테마 지정 | 업로드 시 |
| **챕터 (Chapters)** | 5단계 내러티브에 기반한 챕터 구간명 초안 | 영상 완성 후 타임스탬프 확정 |
| **자막** | 한국어 자막 필수 + 영어 자막 권장 | 업로드 후 |
| **카드/엔드스크린** | 연결할 이전 영상 또는 플레이리스트 지정 | 업로드 시 |

**설명(Description) 구조 템플릿:**

```
{한 줄 메시지를 변형한 호기심 유발 문장 — 검색 결과 노출용, 150자 이내}

📌 이 영상에서 다루는 내용:
{핵심 질문 3개를 자연어로 변환}

⏱ 챕터:
00:00 {HOOK 주제어}
00:30 {SECTION01 주제어}
{...영상 완성 후 타임스탬프 확정}

📚 참고 자료:
- {직접 언급 학자의 대표 저서}
- {리서치 출처 중 시청자에게 유용한 것}

🔗 해빛 채널:
- 구독: {채널 URL}
- 플레이리스트: {시리즈 URL}

#해시태그1 #해시태그2 #해시태그3
```

**태그(Tags) 전략:**

> 카테고리가 Education이므로, 태그와 설명에서 기술 키워드를 적극 포함하여 Science & Technology 검색 트래픽도 흡수한다.

| 구분 | 개수 | 예시 |
|------|------|------|
| **메인 키워드** | 3개 | 주제 핵심 검색어 (높은 검색량) |
| **롱테일 키워드** | 7~10개 | 세부 질문형 검색어 (낮은 경쟁) |
| **기술 커버 태그** | 3~5개 | AI, 인공지능, ChatGPT 등 기술 도메인 검색어 |
| **채널 공통 태그** | 3개 | 해빛, AI철학, 기술인문학 |

**썸네일(Thumbnail) 전략:**

| 요소 | 기획 시 결정 |
|------|------------|
| **비주얼 요소** | 비주얼 모티프 또는 키 비주얼 Shot 기반 (TITLECARD Shot과 연동) |
| **텍스트** | 제목의 핵심 키워드 2~3단어 (제목과 동일 금지 — 보완 관계) |
| **감정 방향** | 호기심 / 놀람 / 긴장 중 택1 (제목 톤과 정합) |
| **색상 전략** | 채널 톤(라벤더·피치 파스텔)과 대비되는 강조색 1개 |

### §13 시의성 & 차별화

**시의성 (왜 지금 이 주제인가):**

| 항목 | 기술 내용 |
|------|----------|
| **뉴스 페그** | 최근 1~2주 내 관련 뉴스·이슈 (있을 때만) |
| **계절/시즌** | 학기 시작, 연말, 새해 등 소비 패턴 연관성 (있을 때만) |
| **트렌드 키워드** | Google Trends / YouTube 검색 추세에서 상승 중인 연관어 |
| **최적 공개 시기** | 위 요소를 종합한 권장 업로드 시점 |

> 시의성이 없는 에버그린 주제는 "에버그린 — 시기 무관"으로 명시.

**경쟁 콘텐츠 분석** — 같은 주제를 다루는 상위 조회수 영상 3개를 분석하고 차별화 포인트를 도출:

| 항목 | 기술 내용 |
|------|----------|
| **경쟁 영상 1~3** | 제목 + 조회수 + 핵심 메시지 요약 (각 1줄) |
| **공통 접근법** | 경쟁 영상들이 공유하는 시각/구조 |
| **차별화 포인트** | 해빛만의 독점 통찰 — "경쟁 영상이 놓친 것" |
| **콘텐츠 유형** | 에버그린 / 트렌딩 / 시리즈파트 |

### §14 Shorts 재활용 전략

롱폼 영상에서 **Shorts 후보 구간 3~5개**를 기획 단계에서 미리 지정한다.

**Shorts 추출점 선정 기준:**

| 우선순위 | 추출 대상 | 이유 |
|---------|----------|------|
| 1 | **HOOK 전체** (≤30초) | 가장 임팩트 있는 도입부 — Shorts 단독 시청 가능 |
| 2 | **텐션 전환점** (메타포 순간) | "아하!" 순간 — 호기심→본편 유도 |
| 3 | **페이오프 핵심 해법** | 실용 정보 — 공유 가치 높음 |
| 4 | **아웃트로 열린 질문** | 토론 유발 — 댓글 참여 유도 |
| 5 | **Song Hook** (HOOK_TYPE: song일 때) | 음악 + 비주얼 — Shorts 최적 포맷 |

각 Shorts 끝에 롱폼으로 유도하는 명시적 CTA 1문장을 기획한다.

### §15 청중 참여 & 리텐션 설계

**댓글 참여 유도 (Comment CTA):**

| 배치 위치 | 질문 유형 | 예시 |
|----------|----------|------|
| **SECTION01 끝** | 공감 질문 — "당신도 이런 경험?" | "혹시 스마트폰 없이 하루를 보내본 적 있으신가요?" |
| **SECTION03 중반** | 선택 질문 — "당신은 어느 쪽?" | "A와 B 중 어떤 방법을 먼저 시도해 보시겠어요?" |
| **OUTRO** | 열린 질문 — 오픈 루프 (기존) | "당신의 오늘은, 어떤 질문으로 시작됩니까?" |

> ⚠️ 댓글 질문은 **양극적 의견**이 나올 수 있는 것이 이상적 (알고리즘 보상). 단, 채널 톤을 벗어나는 논쟁 유도는 금지.

**구독 유도 (Subscribe CTA):**

| 배치 위치 | 형식 | 톤 |
|----------|------|-----|
| **HOOK 직후** (30초 지점) | "이런 질문이 궁금하다면, 구독과 알림 설정" | 가볍고 빠르게 — 1문장 이내 |
| **OUTRO 직전** | "다음 주제 예고 + 구독 권유" | 다음 영상 호기심 연결 |

> ⚠️ 구독 CTA는 대본 전체에서 최대 2회. 과도한 CTA는 시청 경험 저해.

**리텐션 절벽 방지 설계:**

| 예상 이탈 구간 | 원인 | 방지 장치 |
|--------------|------|----------|
| **HOOK 끝 (30초)** | "기대한 내용이 아닌데" | 30초 이내 영상 로드맵 제시 — "오늘은 3가지를 다룹니다" |
