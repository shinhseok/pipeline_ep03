# 서사 강화 원칙 (§4)

> script-director.md의 §4 참조 파일. 대본 작성 시 이 원칙을 적용한다.

---

스틸 컷 이미지로 연결되는 영상이므로, 대본 자체가 '이야기'의 흐름을 가져야 한다.
시청자가 정보를 듣는 것이 아니라 **여정을 함께 걷는** 경험을 설계한다.

### 4-0. 통찰의 층위 — 대본의 깊이 설계 ⚠️ (최우선)

**대본 작성 전, 소재에서 꺼낼 수 있는 통찰을 3개 층위로 분리한다.**

```
표층    누구나 이미 아는 이야기       ← 여기서 멈추면 식상한 영상
중층    한 발 더 들어간 분석          ← 흥미롭지만 예상 범위
심층    일상을 다르게 보게 하는 전환   ← 여기까지 가야 함
```

**실행 규칙**:

1. **PHASE A 시작 전, 3층위 테이블을 명시적으로 작성**한다
   - 표층: 이 소재로 누구나 할 수 있는 이야기는?
   - 중층: 한 겹 벗기면 어떤 구조/패턴이 보이는가?
   - 심층: 시청자가 영상 끝나고 자기 일상을 다르게 볼 수 있는 전환점은?

2. **심층 통찰이 이야기에서 자연스럽게 올라와야 한다**
   - 논문 결론처럼 선언하지 않는다
   - 구체적 사례·장면이 쌓이면서 시청자가 "아, 그렇구나"를 느끼는 구조
   - S01에서 씨앗 → S02에서 패턴 인식 → S03에서 통찰 귀결

3. **모든 소재를 통찰 축에 비춰 판정**한다
   - 각 소재가 심층 통찰에 기여하는가? → 유지
   - 흥미롭지만 축에서 벗어나는가? → 삭제
   - 한 섹션에 소재 4개 이상이면 과적 경고 → 선별

**예시** (산업혁명 + AI 노동):
```
표층    "AI가 일자리를 뺏는다"                    ← 누구나 아는 불안
중층    "기술이 노동의 방식·리듬을 바꾼다"          ← 흥미롭지만 예상 가능
심층    "기술이 바꾸는 건 일이 아니라,              ← 일상의 시선이 달라짐
        인간을 바라보는 시선이다"
```

→ 장인이 '노동력'이 되고, 아이가 '싼 팔'이 되는 **구체적 장면**에서 통찰이 올라옴
→ "그거 AI가 더 잘하지 않아?"라는 **일상 대화**에서 같은 패턴을 인식

**체크**: 대본 완성 후 핵심 문장을 뽑아 "이건 표층인가, 중층인가, 심층인가?" 자문한다.
표층·중층에 머물러 있으면 재구성한다.

### 4-0-1. 물음표 검토 — 처음 보는 시청자의 눈 ⚠️ (필수)

> 이 검토는 4-20(초등학생 이해도)과 다르다. 4-20은 문장 단위 명확성, 물음표 검토는 **이야기 흐름의 논리적 연결**이다.

**검토 항목**:

1. **소재 전환 시 다리가 있는가** — 시대·장소·주제 전환에 연결 고리 확인
2. **한 섹션에 소재가 과적되지 않았는가** — 독립 소재 4개 이상이면 선별
3. **지시어·대명사의 대상이 명확한가** — "이번엔", "그것" 등 3초 안에 특정 가능한가
4. **통찰 축에서 이탈한 곁가지가 없는가** — "빼면 메시지가 약해지나?" No면 삭제

**출력**: `FLOW_CHECK` 블록으로 물음표 지점 + 해소 방법 기록

### 4-0-2. 질문 체인으로 통찰 도달 ⚠️ (필수)

**통찰을 직접 진술하지 않는다. 질문이 심화되면서 시청자가 따라가며 도달한다.**

```
Q1: 현상    "왜 이런 일이 일어날까"
Q2: 원인    "~때문인 건가?"
Q3: 자기    "나도 그런 건 아닌가"
Q4: 재구성  "그렇다면 진짜 질문은 뭘까"
→ 통찰: 질문 재구성에서 자연스럽게 도달
→ 행동: 통찰에 따른 구체적 행동 변화 1비트
```

**핵심 규칙**:
- 직접 진술("사람을 쓸모로 보게 된다")은 시청자의 발견 기회를 뺏는다
- 질문이 깊어지는 과정 자체가 서사다
- Q3(자기에게 돌아옴)이 반드시 포함되어야 통찰이 체감된다
- Q4(재구성) 후 학자·레퍼런스가 프레임을 제공하면 자연스러움

**검증**: SEC03에서 통찰이 등장하는 대목을 뽑아 "이건 화자가 직접 말한 건가, 질문의 귀결로 도달한 건가?" 자문한다.

### 4-0-3. 메시지-착지 정합성 검증 ⚠️ (필수)

**착지 문장(OUTRO/마지막 대사)이 핵심 메시지의 정확한 응답인지 검증한다.**

```
검증 질문: "이 착지 문장이, 핵심 메시지가 제기한 문제에 대한 응답인가?"

❌ 메시지: "기술이 사람을 쓸모로 보게 한다"
   착지: "내 손으로" → 수단의 선택 (다른 축)

✅ 메시지: "기술이 사람을 쓸모로 보게 한다"
   착지: "이름을 먼저 불러야겠다" → 시선의 전환 (같은 축)
```

**체크리스트**:
1. 핵심 메시지를 한 문장으로 뽑는다
2. 착지 문장을 한 문장으로 뽑는다
3. 두 문장이 같은 축(같은 문제-응답 관계)에 있는가?
4. 착지가 메시지와 다른 축에 있으면, 착지를 바꾸거나 메시지를 재정의한다

**출력**: 대본 끝 `MESSAGE_CHECK`에 착지 정합성 검증 결과를 추가한다

### 4-1. 이야기 구조로 설계하기

5단계 내러티브 구조를 단순 정보 전달이 아닌 이야기 구조에 얹는다.

```
훅 (Hook)        → 사건의 시작: 무언가 이상하다, 질문이 생긴다 (≤30초, 핵심 8초)
셋업 (Setup)     → 배경 탐색: 이게 왜 생겼는지 과거로 거슬러 올라간다
텐션 (Tension)   → 핵심 발견: 아, 이런 구조였구나 (메타포로 본질을 드러냄)
페이오프 (Payoff) → 귀환: 이제 무엇을 다르게 할 수 있는가
아웃트로 (Outro)  → 여운: Hook 순환 + 오픈 루프 질문 (20~30초)
```

### 4-2. 구체적 장면으로 시작하기 (Show, Don't Tell)

추상적 선언 대신 구체적 장면·상황으로 시작한다.

```
❌ "AI가 인간의 사고를 바꾸고 있습니다."
✅ "아침에 눈 뜨자마자, 손이 먼저 폰으로 향했어요. 아직 눈도 제대로 못 떴는데."
```

### 4-3. 시청자를 이야기 안으로 끌어들이기

'당신'과 '우리'를 이야기의 주인공으로 만든다.
추상적 '사람들' 대신 시청자가 직접 경험했을 장면을 소환한다.

```
❌ "현대인들은 기술에 의존하게 되었습니다."
✅ "검색창 없이 길을 찾아본 게 언제였는지, 기억나시나요?"
```

### 4-4. 섹션 간 이야기 이어받기 (Narrative Thread)

각 섹션은 이전 섹션의 이야기를 이어받아 발전시킨다.
Hook에서 던진 질문이나 이미지가 Outro에서 새로운 빛으로 다시 등장해야 한다.

```
Hook의 씨앗 → SECTION 01~03에서 키워짐 → Outro에서 다른 빛으로 귀환
```

### 4-5. 긴장과 해소 설계

영상 전체에 긴장 곡선을 설계한다.

```
Hook          → 긴장 씨앗: 의문, 불편함, 호기심
SECTION 01   → 긴장 고조: "이게 왜 이렇게 됐을까" 배경 탐색
SECTION 02   → 전환점: 메타포가 등장하며 '아하!' 순간 발생
SECTION 03   → 해소: 시청자가 선택할 수 있는 길이 생긴다
Outro         → 여운: 답이 아닌 새로운 질문으로 마무리
```

### 4-6. 학자의 이론을 이야기 속 목소리로

학자의 이름을 직접 꺼낼 때도, 이론 설명이 아닌 이야기 속 장면으로 등장시킨다.

```
❌ "마셜 매클루언은 미디어는 메시지라고 주장했습니다."
✅ "마셜 매클루언이라는 학자가 있었어요. 한번은 이런 말을 했어요.
   '무엇으로 말하는지가, 무엇을 말하는지보다 더 많은 걸 바꾼다'고요.
   처음엔 무슨 말인지 몰랐어요. 그런데..."
```

### 4-7. Hook 오프닝: 영화 첫 장면 규칙 ⚠️ (필수)

역사적 에피소드를 Hook으로 사용할 때, 날짜 + 장소 + 인물 상황으로 영화 첫 장면처럼 시작한다.
역사 서술조가 아닌, 독자가 그 장면 안에 있는 것처럼 현장감 있게 묘사한다.

```
❌ "오늘로부터 약 600년 전, 독일의 작은 도시에서 한 남자가 쇠를 녹이고 있었습니다."
   (역사 서술조 — 거리감 있음)

✅ "1450년, 독일의 작은 도시 마인츠.
   한 남자가 작업실 문을 꼭 잠갔어요. 빚쟁이들이 밖에서 기다리고 있었거든요."
   (날짜+장소+인물 심리 — 현장감 있음, 인물에 감정이입 유도)
```

인물의 심리·두려움·동기를 구체적으로 묘사하여 시청자가 감정이입할 수 있게 한다.

### 4-8. 관통 캐릭터 원칙 ⚠️ (필수)

Hook에서 등장한 역사 인물이나 이미지는 영상 전체를 꿰뚫는 관통 캐릭터가 되어야 한다.
Hook에서만 잠깐 쓰고 사라지는 것을 금지한다.

```
✅ 올바른 관통 구조:
  Hook     → 구텐베르크가 '문을 잠근다' (상황 설정)
  SEC 01   → 구텐베르크 이야기로 자연스럽게 이어감
  SEC 03   → 구텐베르크를 다시 소환하여 교훈을 연결
  OUTRO    → Hook의 '잠긴 문' 이미지가 '여는 문'으로 시각적으로 귀환

❌ 금지 구조:
  Hook     → 구텐베르크 등장
  SEC 01~03 → 구텐베르크 완전히 사라짐, 별개 정보 나열
  OUTRO    → 맥락 없이 새 문장으로 마무리
```

### 4-9. 섹션 전환 훅 ⚠️ (필수)

각 SECTION 끝에 다음 SECTION으로 자연스럽게 끌어당기는 전환 질문 1개를 배치한다.
전환 질문이 없으면 시청자는 영상을 이탈할 수 있다.

```
✅ SECTION 01 끝 전환 훅 예시:
  "...그렇다면 AI는, 우리를 어떤 색으로 물들이고 있을까요?"
  (→ SECTION 02의 AI 메타포 분석으로 자연스럽게 연결)

✅ SECTION 02 끝 전환 훅 예시:
  "포스트먼은 이걸 알면서도 기술을 거부하지 않았어요. 그는 더 현명한 방법을 제안했어요."
  (→ SECTION 03의 실천 제언으로 자연스럽게 연결)
```

### 4-10. 학자 등장 3단계 구조 ⚠️ (필수)

학자를 직접 언급할 때, 이름만 던지는 것이 아니라 3단계 구조로 전개한다.

```
[1단계] 학자 소개: 이름 + 시대 배경 + 맥루언과의 관계 등 간결한 맥락
[2단계] 구체적 일상 예시: 학자의 이론을 자동차·TV 등 시청자가 아는 예로 먼저 보여줌
[3단계] AI에 적용: "AI는 어떨까요?"로 넘어가 2단계 예시를 현재에 연결

❌ 금지 패턴:
  학자 이름 → 이론 한 줄 요약 → 다음 내용으로 넘어감

✅ 올바른 패턴:
  학자 소개 → 자동차 예시로 이론 체감 → "AI는 어떨까요?"로 전환
```

### 4-11. 실천 제언 네이밍 규칙

SECTION 03의 실천 방법론에는 기억에 남는 고유 이름을 부여한다.
막연한 행동 지침이 아닌, 시청자가 다음날도 기억할 수 있는 개념 이름으로 포장한다.

```
❌ 막연한 지침: "기술과 자신 사이에 작은 심리적 거리를 두세요."
✅ 네이밍 지침: "포스트먼은 이런 사람이 되어야 한다고 했어요. '애정 어린 저항군'이요."
```

### 4-12. 사유의 여백(PAUSE) 설계 ⚠️ (필수)

나레이션 중간에 시청자가 스스로 생각에 잠길 수 있는 침묵 구간을 명시적으로 설계한다.
`[PAUSE: N초]` 태그를 문장 사이에 삽입한다.

```
✅ PAUSE 삽입 권장 지점:
  - 핵심 질문을 던진 직후: "~없으신가요? [PAUSE: 2초]"
  - 메타포의 핵심 이미지가 떠오른 직후: "잉크가 퍼집니다. [PAUSE: 1.5초]"
  - 거래의 대가를 언급한 직후: "그 사실조차 잊어버렸습니다. [PAUSE: 1.5초]"
  - OUTRO 마지막 질문 직전: 최소 2초 침묵

❌ 금지:
  - 영상 전체에 PAUSE가 0개 (반드시 4~6개 이상 삽입)
  - 설명 문장 중간에 무의미한 PAUSE 삽입
```

### 4-13. 정서적 공명 원칙 ⚠️ (필수)

정보를 나열하듯 전달하지 않는다. 시청자가 자신의 일상을 투영하여 '아, 나도 그랬지'라는
감정적 울림을 느끼도록 문장을 설계한다.

```
✅ 정서적 공명 기법:
  - 감각적 비유: "지저분하고 축축한 과정의 시간", "덜덜 떨리는 마음으로"
  - 명사형 나열로 여운: "틀려볼 수 있는 시간. 헤매는 고독한 침묵. 느리고 지저분한 과정."
  - 2인칭 직접 호명: "당신의 사유가 뿌리 내릴 토양은 메말라가고 있습니다."
  - 질문 후 시각적 침묵: 섹션 마지막 질문 후 나레이션 없이 BGM만 흐르는 5~7초 구간 설계

❌ 금지:
  - "~라고 합니다", "~라고 알려져 있습니다" 등 교과서 서술 어미
  - 시청자가 자기 경험을 투영할 수 없는 추상적 진술만 나열
```

### 4-14. 즉각적 실천 권유 ⚠️ (필수)

SECTION 03의 실천 제언에서는 반드시 "지금부터는"으로 시작하여 시청자가 즉각적으로
행동에 옮길 수 있도록 권유한다. 미래로 미루는 표현은 절대 금지한다.

```
❌ "내일부터는 이렇게 해보세요."
❌ "다음에는 한번 시도해 보세요."
✅ "지금부터는 이렇게 한 번 해보세요."
✅ "오늘, 이 영상이 끝나면 바로 해보세요."
```

### 4-15. 페이오프(SEC03) 응집력 규칙 ⚠️

**사례 나열 금지 — 하나의 관통 문장으로 수렴**

SEC03에 복수의 사례(역사적 해결책, 현대 사례, 학자 원칙 등)를 배치할 때,
독립적 "답"으로 나열하지 않고 **하나의 관통 문장**이 모든 사례를 꿰뚫게 한다.

```
❌ 나열 구조: 답1(공장법) → 답2(체로키) → 답3(포스트만) → 질문
✅ 수렴 구조: 관통 진실 선언 → 사례가 이 진실을 증명 → 시청자 행동으로 귀환
```

**관통 문장 작성법**:
- 모든 사례가 공유하는 **하나의 행동 원칙**을 먼저 추출
- 이 원칙을 SEC03 초반에 선언 → 이후 사례는 이 원칙의 "증거"로 배치
- SEC03 끝에서 이 원칙을 시청자의 행동으로 전환

**예시**: 공장법 + 체로키 + 포스트만 → 관통 진실: "질문을 멈추지 않은 사람들이 규칙을 바꿨다"
→ 공장법 = 질문한 노동자들의 결과 / 체로키 = 현대판 질문 / 포스트만 = 이 태도에 이름을 붙여준 사람

### 4-16. 모티프 전구간 관통 ⚠️

Hook(Song Hook 포함)에서 심은 핵심 모티프(이미지·단어)는 SEC01~SEC04 **전 구간**에서 변형·반복한다.

```
❌ Hook "손" 모티프 → SEC01~02만 사용 → SEC03~OUTRO에서 소멸
✅ Hook "손끝" → SEC01 "실 잣는 손" → SEC02 "생각 감아 올림"
   → SEC03 "질문하는 손" → OUTRO "같은 질문을 쥔 손"
```

- 모티프가 **SEC03에서 반드시 재등장**해야 페이오프의 감정적 무게가 살아남
- OUTRO에서 모티프가 Hook과 **대구(對句)**를 이루며 순환 완성
- Song Hook 마지막 줄과 OUTRO 마지막 줄이 같은 모티프로 호응해야 함

### 4-17. SEC03 → OUTRO 유기적 전환 ⚠️

SEC03 마지막 이미지/단어가 OUTRO 첫 문장으로 자연스럽게 이어져야 한다.
감정선이 끊기면 시청자가 OUTRO에서 이탈한다.

```
❌ SEC03: "어느 쪽을 선택하시겠어요?" → OUTRO: "방적공과 당신." (감정선 단절)
✅ SEC03: "당신의 손으로 질문해 보세요." → OUTRO: "물레를 돌리던 손, 키보드를 두드리는 손." (모티프 연결)
```

- SEC03 마지막 문장의 핵심 이미지를 OUTRO 첫 문장이 받아서 확장
- 새로운 개념이나 이미지를 OUTRO에서 갑자기 도입하지 않음

### 4-18. 학술 용어 직관화 + 각주 원어 병기 ⚠️

학자의 고유 표현이 일상에서 쓰지 않는 조합일 때:
1. 근본 의미를 보존하면서 **직관적 한국어로 치환**
2. Fact Check 표에 **원어를 각주 병기**

```
❌ "스스로를 '애정 어린 저항운동가'라 불렀죠." (낯선 조합 — 시청자가 3초 안에 뜻을 못 짐작)
✅ "스스로를 '다정한 반항아'라 불렀죠." + Fact Check에 원어 "loving resistance fighter" 병기
```

**판단 기준**: "이 표현을 처음 듣는 중학생이 3초 안에 뜻을 짐작할 수 있는가?"
- 짐작 가능 → 원어 그대로 사용
- 짐작 불가 → 직관적 치환 + 각주 원어

### 4-19. 서사 수렴 원칙 ⚠️ (최우선)

**대본의 서사는 진행될수록 한 점으로 수렴해야 한다.**

SEC01~SEC03의 모든 사례·메타포·학자·개념은 최종적으로 **하나의 질문 또는 문장**으로 모여야 한다. SEC03 끝에서 이 한 점이 명시적으로 드러나고, OUTRO에서 같은 한 점이 순환해야 한다.

```
❌ 펼쳐짐: SEC03에서 개념 A, 개념 B, 질문 C가 나열 → 어디로 향하는지 모호
✅ 수렴:   모든 사례가 하나의 질문으로 모임 → OUTRO에서 같은 질문이 순환

예시: 공장법(질문이 법을 만듦) + 체로키(선을 그음) + 포스트만(질문을 멈추지 않는 사람)
  → 한 점: "이것이 나를 돕고 있나, 대체하고 있나"
  → OUTRO: "200년의 거리를 두고, 같은 질문을 쥐고 있습니다. 이것이 나를 돕고 있나, 대체하고 있나."
```

**체크 방법**: SEC03의 마지막 핵심 문장을 뽑아서, SEC01~SEC03의 모든 요소가 이 문장을 향해 좁혀지고 있는지 역추적한다. 한 요소라도 이 문장과 무관하면 삭제하거나 연결을 보강한다.

### 4-20. 초등학생 이해도 검증 ⚠️

**처음 보는 사람이 머릿속에 그림을 그릴 수 있는가?**

대본 완성 후, 모든 문장을 "이 영상을 처음 보는 초등학생에게 읽어줬을 때 머릿속에 장면이 그려지는가?" 기준으로 점검한다.

**주요 검증 항목**:
1. **메타포에 다리가 있는가**: 추상적 메타포(지도, 나침반, 우상 등)를 사용할 때, 왜 그것이 그 메타포인지 한 문장 이상의 근거가 선행되는가
2. **사례 간 연결고리가 명시되는가**: 서로 다른 시대·장소의 사례를 나열할 때, 공통 원리가 한 문장으로 명시되는가
3. **전문 용어 없이 전달되는가**: 직공→실 짜는 사람, 방적기→증기기관이 달린 물레 등 일상 단어로 교체
4. **'왜 유령인가'류의 빈 설명이 없는가**: 새로운 개념·용어 등장 시 즉시 한 마디로 풀이

```
❌ "시급 2달러의 '유령 노동'."  (왜 유령? 설명 없음)
✅ "시급 2달러. 누구도 모르게 일하니까 '유령 노동'이라 불러요."

❌ "이 역사가 우리에게 남긴 지도예요."  (왜 지도? 비약)
✅ "앞서 걸어간 사람들이 남긴 지도예요."  (발자국=지도, 직관적)
```

### 4-21. 단정적 자각 유도 금지 ⚠️

시청자에게 "당신도 같은 상황이다"라고 **단정하지 않는다**. 스스로 생각해볼 수 있도록, 질문이나 넌지시 느낌을 주는 표현을 사용한다.

```
❌ "지금 '나는 괜찮다'고 생각하는 여러분도 같은 흐름 안에 있습니다."  (단정)
✅ "혹시 지금, 여러분도 '나는 괜찮다'고 느끼고 있진 않은지요."  (질문)
```

### 4-22. 오디오 스토리텔링 원칙 ⚠️ (라디오 대본 기준)

**귀로만 들어도 이야기가 그려져야 한다.**

대본은 영상 없이 나레이션만으로도 이해될 수 있어야 한다. 아래 5개 하위 규칙을 STEP 03 편집 시 적용한다.

**A. 숫자 다이어트**
- 한 문단에 숫자 최대 2개. 3개 이상 나열되면 관계적 언어로 치환
- 핵심 대비(5만→300)만 남기고, 보조 숫자(45kg, 8시간, 17년)는 삭제
```
❌ "면화 45kg을 실로 만드는 데 5만 시간. 매일 8시간씩 17년. 300시간으로 줄었습니다."
✅ "면화를 실로 만드는 데, 5만 시간이 걸렸어요. 300시간으로 줄었습니다."
```

**B. 컨텍스트 브릿지**
- 시대·장소·주제가 전환될 때 한 문장 다리를 놓는다
- 특히 200년 이상 시간 점프 시 "200년이 지났습니다" 류의 명시적 브릿지 필수
```
❌ (아동노동 묘사) → (바로 유령노동)  ← 시대 점프 인식 불가
✅ (아동노동 묘사) → "200년이 지났습니다." → (유령노동)
```

**C. 한 문장 한 아이디어**
- 복합문(A했고, B하고 있어요)을 단문 2개로 분리
- 귀로 들을 때 한 문장에 정보가 2개 이상이면 뒤의 것을 놓침
```
❌ "증기기관은 생산성을 줬지만 삶의 리듬을 빼앗았고, AI는 효율을 주면서 생각하는 힘을 가져가고 있어요."
✅ "증기기관은 생산성을 줬지만, 삶의 리듬을 빼앗았어요. AI는 효율을 주면서, 생각하는 힘을 가져가고 있어요."
```

**D. 법률·연표 서술화**
- 연도+법률명 나열은 귀로 구분 불가 → 내용 서술로 전환
```
❌ "1833년 공장법, 1842년 광산법, 1847년 10시간법."
✅ "아이들을 공장에서 내보내는 법, 탄광에서 내보내는 법, 하루 열 시간만 일하게 하는 법."
```

**E. 학자 인용 3박자**
- ① 맥락 진입 → ② 핵심 질문/사상 직접 인용 → ③ 일상 버전으로 전환
- 학자의 철학적 질문과 시청자의 일상 질문을 분리하되 연결
```
✅ "복잡한 분석이 필요한 게 아니래요." (맥락)
   → "이 기술 때문에 우리는 무엇을 얻고, 무엇을 잃고 있는가." (핵심)
   → "이 AI가 내 손을 돕고 있나, 대체하고 있나." (일상 버전)
```

### 4-23. 코다 설계 — 완만한 하강 ⚠️ (필수)

**클라이맥스 후 급착륙을 금지한다. 음악의 코다처럼 완만하게 하강하며 여운을 만든다.**

```
구조: 클라이맥스(fortissimo) → 코다(piano) → 여운(pianissimo)

클라이맥스: 핵심 착지 문장 (유니즌, 선언, 질문 등)
코다:       조용한 일상 동작 1비트 — SEC01 도입부의 원형 회귀
여운:       OUTRO — 프레임 나레이터가 천천히 닫음 + [PAUSE]
```

**코다 작성법**:
- SEC01 도입에서 등장한 일상 이미지를 변형·반복한다 (원형 회귀)
- 거창한 결의가 아니라 작은 동작으로 "내일"을 보여준다
- 클라이맥스와 OUTRO 사이에 [PAUSE: 2초] 이상 배치

```
❌ 급착륙:
  SEC03 착지: "이름을 먼저 불러야겠다."
  → 바로 OUTRO: "다음 이야기도~"

✅ 완만한 하강:
  SEC03 착지: "이름을 먼저 불러야겠다."
  → 코다: "양초를 끄기 전에, 창가 자리를 닦아두었다." (일상 동작)
  → [PAUSE: 2초]
  → OUTRO: "200년의 거리를 두고, 같은 밤을 보내고 있었어요."
```

**검증**: 클라이맥스 직후 OUTRO로 바로 가는지 확인. 사이에 코다 비트가 없으면 추가한다.

### 4-24. 교차 서사 원칙 — 병렬 화자/시점 설계 ⚠️ (조건부)

**두 개 이상의 독립 화자·시점·시간대를 교차 편집으로 엮는 서사 형식.**
일기 교차, 편지 교차, 독백 교차, 시점 교차 등 다양한 변형에 적용한다.
`02_planning`에서 교차 서사 형식을 채택했을 때만 적용.

#### A. 순수 교차 원칙 (Pure Cross-Cutting)

화자들은 서로를 알지 못한다. 편집이 대화를 만든다.

```
❌ 화자 B가 화자 A의 기록에 직접 접근 — "200년 전 일기를 다시 폈다"
❌ 화자 A가 미래를 예언 — "먼 훗날 기계가 우리 자리를 차지하겠지"
✅ 각자 독립적으로 쓰되, 편집 배치가 의미를 생성 (쿨레쇼프 효과)
```

- 각 화자는 **자기 시대·상황 안에서만** 생각하고 쓴다
- 교차 배치에서 의미가 발생하는 것이지, 화자가 의미를 말하는 게 아니다
- 시청자만 양쪽을 함께 보며 패턴을 인식한다

#### B. 전환 유형 교대 (Transition Alternation)

교차 지점마다 전환 유형을 명시하고, 같은 유형 3연속을 금지한다.

| 전환 유형 | 정의 | 예시 |
|-----------|------|------|
| **Mirror** | 같은 행동·감정, 다른 시대 | A "실을 잣는다" ↔ B "키보드를 친다" |
| **Contrast** | 반대되는 상황 | A "기계가 사람을 이기겠나" ↔ B "AI가 나보다 빨라" |
| **Escalation** | 긴장·통찰이 한 단계 상승 | A의 불안 → B의 더 큰 불안 |

```
✅ Mirror → Contrast → Escalation → Mirror (교대)
❌ Mirror → Mirror → Mirror (동일 3연속 — 단조)
```

#### C. 세그먼트 리듬 (Progressive Shortening)

교차 세그먼트 길이를 **점진적으로 줄여** 몰입도를 높인다.

```
SEC01: 4~5문장 단위 → SEC02: 3~4문장 → SEC03: 1~2문장 (핑퐁)
```

- 초반은 각 화자의 세계를 충분히 보여줌
- 후반으로 갈수록 빠르게 전환하며 긴장감 상승
- 클라이맥스에서 **근접 유니즌**(같은 뜻, 다른 말)으로 수렴

#### D. Dramatic Irony 활용

시청자만 아는 정보를 의도적으로 배치한다.

```
화자 A(1764): "기계가 사람 손을 이기겠나."
→ 시청자는 결과를 알고 있다 (산업혁명)
→ 이 간극이 감정적 긴장을 만든다
```

- 과거 화자가 미래를 모르는 것 자체가 서사 장치
- 현재 화자가 미래를 모르는 것도 동일 패턴 ("AI가 내 자리를 대체할까" → 시청자도 모름 = 공감)

#### E. 검증 체크리스트

대본 완성 후 교차 서사 형식일 때 추가 검증:

1. **순수 교차 위반 없음**: 화자 간 직접 접근·인용·예언 0건
2. **전환 유형 교대**: 같은 유형 3연속 없음
3. **세그먼트 축소**: SEC01→SEC03으로 갈수록 단위 줄어듦
4. **근접 유니즌**: 클라이맥스에서 화자들이 같은 뜻을 다른 말로 수렴
