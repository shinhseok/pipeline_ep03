# LLM 폴리싱 규칙

> script-director.md §8.5 참조 파일. 대본 최종본 완성 후 LLM 특유 표현을 제거한다.
> 이 목록은 지속 업데이트한다 — 새로운 패턴 발견 시 추가.

---

## 목적

LLM이 생성한 텍스트에는 인간 작가가 쓰지 않는 특유의 패턴이 있다.
해빛 대본은 **구어체 나레이션**이므로, 이러한 패턴이 남아 있으면 시청자가 "AI가 쓴 글"로 인식한다.

---

## 한국어 LLM 특유 표현 목록

### 1. 과잉 탐구 표현

| LLM 표현 | 처리 |
|----------|------|
| "~에 대해 깊이 들여다보면" | 삭제 후 직접 서술 |
| "~를 자세히 살펴보면" | 삭제 후 직접 서술 |
| "~를 탐구해 봅시다" | 삭제 (해빛는 "탐구하자"고 말하지 않음) |
| "~를 파헤쳐 보겠습니다" | 삭제 |

### 2. 과잉 강조·감탄

| LLM 표현 | 처리 |
|----------|------|
| "흥미롭게도" | 삭제 (사실 자체가 흥미로우면 수식어 불필요) |
| "놀랍게도", "놀라운 사실은" | 삭제 |
| "주목할 만한 점은" | 삭제 |
| "중요한 것은" | 삭제 또는 구어체 전환 ("핵심은요") |
| "특히 주목해야 할 것은" | 삭제 |
| "매우 흥미로운" | "흥미로운" 또는 삭제 |

### 3. 과잉 나열·일반화

| LLM 표현 | 처리 |
|----------|------|
| "다양한" (맥락 없이) | 구체적 수량·예시로 교체, 또는 삭제 |
| "여러 가지", "수많은" | 구체적 수량으로 교체 |
| "~등이 있습니다" | 직접 나열 후 마침 |
| "~뿐만 아니라 ~도" (3회+ 반복) | 구조 재편 |

### 4. 헤징 (회피·약화)

| LLM 표현 | 처리 |
|----------|------|
| "~라고 할 수 있습니다" | 단정형으로 전환 ("~입니다", "~이에요") |
| "어쩌면 ~일지도 모릅니다" | 확신이 없으면 삭제, 있으면 단정 |
| "어느 정도는" | 삭제 |
| "일종의" | 삭제 또는 구체화 |

### 5. 번역투·문어투

| LLM 표현 | 처리 |
|----------|------|
| "~의 영역에서" | "~에서" |
| "~의 맥락에서" | "~에서", "~으로 보면" |
| "~의 관점에서 보면" | "~으로 보면" |
| "~에 있어서" | "~에서" |
| "~를 통해" (3회+ 반복) | 2회 이하로 축소 |
| "기반으로 하여" | "바탕으로" |

### 6. 접속 상투어

| LLM 표현 | 처리 |
|----------|------|
| "그러나" (3회+) | 1~2회로 축소, 나머지는 접속사 없이 연결 |
| "따라서" | "그래서" 또는 삭제 |
| "결론적으로" | 삭제 (OUTRO는 결론을 선언하지 않음) |
| "이러한 이유로" | 삭제 또는 "그래서" |
| "한편" (2회+) | 1회로 축소 |

### 7. AI 메타 표현 (페르소나 파괴)

| LLM 표현 | 처리 |
|----------|------|
| "이 질문에 답하기 위해" | 삭제 |
| "살펴보겠습니다" | 삭제 |
| "알아보겠습니다" | 삭제 |
| "정리하면" | 삭제 |
| "요약하자면" | 삭제 |
| "마무리하며" | 삭제 (해빛는 "마무리한다"고 선언하지 않음) |

---

## 폴리싱 절차

1. 최종본 완성 후 위 7개 카테고리를 순회하며 패턴 검출
2. 검출된 표현을 교체 또는 삭제
3. 교체 후 문장 자연스러움 재확인 (삭제로 문맥이 끊기지 않는지)
4. 결과를 `LLM_POLISH_LOG`로 출력

### LLM_POLISH_LOG 형식

```
LLM_POLISH_LOG:
  검출: {N}건
  수정: {목록 — "흥미롭게도 → 삭제", "다양한 → 세 가지" 등}
  잔여: 0건
```
