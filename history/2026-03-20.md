# 2026-03-20 작업 기록

## 1. NB2 프롬프트 최적화 실험 (9종)

### 실험 목적
ref_images(턴어라운드 시트)가 제공될 때, flow_prompt에서 어떤 요소를 제거할 수 있는지 체계적으로 검증.

### 실험 결과 요약

| 실험 | 주제 | 결론 |
|------|------|------|
| A | 기존 프롬프트 (기준선) | shot12 기준 이미지 생성 |
| B | 외형 묘사 제거 | ref 이미지가 외형 담당 → 텍스트 묘사 불필요 확인 |
| C | 스타일 텍스트 제거 | ref 이미지 스타일이 지배 → 스타일 묘사 불필요 (단, style_ref 없으면 필요) |
| D | REDRAW 제거 | THIS {name} 라벨이 REDRAW 역할 대체 |
| E1~E3 | style_ref 효과 검증 | style_ref 첨부 시 ref 없는 shot도 스타일 일관성 유지. 캐릭터 포함 이미지는 오염 발생 → 캐릭터 없는 이미지만 사용 |
| F | THIS {name} 네이밍 | 서수 참조("첫 번째 이미지의~") 대비 직관적이고 정확. 14개 ref까지 제어 성공 |
| G | 캐릭터 턴어라운드 시트 | 4뷰(정면/3·4/측면/후면) 시트로 캐릭터 재현 정확도 향상. 넥커치프→셔츠 오인식 해결 |
| H | 소품 턴어라운드 시트 | 4뷰(정면/측면/상면/3·4) 시트로 소품 구조 정확도 향상 |
| Q | 복수 캐릭터+소품 통합 | 3캐릭터+3소품+스타일 ref = 14개 동시 참조 성공 |
| R | 카운트 제약 제거 | 구도 묘사에서 요소가 명시되므로 별도 카운트 불필요 |
| S | 한글 vs 영문 프롬프트 | 한글이 품질+속도 모두 우세 (생성시간 32% 빠름, 채색 제어 더 깔끔) |

### 핵심 발견

1. **ref 이미지 스타일 지배**: NB2는 ref 이미지의 드로잉 스타일을 텍스트보다 우선 적용
2. **THIS {name} 라벨**: "THIS {name} — 이 대상의 형태만 따라 그려줘:" 통일 라벨이 서수 참조보다 효과적
3. **style_ref 필수**: 캐릭터 없는 앵커 이미지를 style_ref로 항상 첨부 → ref 없는 shot도 스타일 일관성
4. **턴어라운드 시트**: 단일 포즈 대비 캐릭터/소품 재현 정확도 대폭 향상
5. **한글 프롬프트 우위**: 의미 밀도 높음, 종결형 단정이 NB2에 더 강한 제약으로 작용

### 실험 산출물

- `experiments/2026-03-20/` — 실험별 폴더 (프롬프트 + 생성 이미지)
- `experiments/2026-03-20/template_scene/` — 확정 템플릿 3종 (씬/캐릭터 시트/소품 시트)

## 2. 파이프라인 개선 (실험 결과 반영)

### 변경 파일 및 내용

#### `anchor-generation.md` — Phase 1 레퍼런스 형식 전환
- 단일 포즈(3/4 뷰 1장) → **턴어라운드 시트**(4뷰 수평 배치)
- 캐릭터: 정면/3·4/측면/후면 + 앞뒤 구분(엉덩이 라인)
- 소품: 정면/측면/상면/3·4뷰
- API content: 캐릭터는 `main.jpeg` 참조, 소품은 `style_ref.png` 참조
- `ref_label`/`ref_file` 필드 추가 (Phase 1 프롬프트에 THIS {name} 라벨 적용)

#### `generate_images.py` — 이미지 생성 스크립트 개선
- V3 라벨을 **`THIS {name} — 이 대상의 형태만 따라 그려줘:`** 통일
- `style_ref` 자동 감지 (`assets/reference/anchor/style_ref.png` 존재 시 자동 첨부)
- Phase 1에서 `ref_label`/`ref_file` 파싱 → 커스텀 라벨로 이미지 생성
- `generate_image()`에 `style_ref_label` 매개변수 추가
- 다중 레퍼런스 경고 한국어화

#### `visual-director.md` — flow_prompt 간소화
- **5단락 → 4단락** (P1 과제소개, P2 배경, P3 구도/감정, P4 채색)
- **THIS {name} 직접 참조** 패턴 도입 (서수 참조 폐기)
- 불필요 요소 제거: 스타일 묘사, 얼굴 규칙, REDRAW, 카운트, 장면 가드
- Self-Reflection 체크리스트 업데이트

#### `variant-templates.md` — Variant 템플릿 업데이트
- Variant A~C를 4단락 + THIS {name} 형식으로 전환
- 응집 패턴 예시 업데이트

#### `sempe-ink.yaml` — 빌딩 블록 폐기 표시
- `v3_task_intro`: "THIS style의 드로잉 스타일로" 형식으로 변경
- 폐기 표시 (레거시 호환 유지): `v3_style_description`, `v3_face_rule`, `v3_redraw_rule`, `v3_single_scene_guard`, `v3_count_template`

#### `CLAUDE.md` — 빠른 참조 업데이트
- 4단락 구조 + THIS {name} 패턴 반영
- API content 배열 구조 설명 추가
- 제거된 항목 명시

### 설계 원칙

**역할 분리**:
- `style_ref` 이미지 → 드로잉 스타일 전달
- 턴어라운드 시트 → 캐릭터/소품 외형 전달
- `THIS {name}` 라벨 → ref와 프롬프트 요소 연결
- `flow_prompt` → 구도, 배치, 크기, 감정, 채색만 담당

**프롬프트 절감 효과**: shot당 ~240자 절감 (스타일 묘사 + 얼굴 규칙 + REDRAW + 카운트 + 장면 가드)
