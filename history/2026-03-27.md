## 1. Gemini API 인증 방식 전환 — API Key → Vertex AI ADC

### 변경 내용
`GEMINI_API_KEY` 기반 인증을 **Vertex AI ADC(Application Default Credentials)** 방식으로 전환.
`gcloud auth application-default login`으로 로컬 인증 후, 코드에서 `genai.Client(vertexai=True, project=..., location=...)` 호출.

### 이유
- API 키 노출 위험 제거 (`.env`에 키 평문 저장 불필요)
- Vertex AI SDK 권장 인증 방식
- 로컬 개발 시 `gcloud auth application-default login` 1회만 실행하면 키 파일 관리 없이 인증

### 설정
- `.env`: `GEMINI_API_KEY` 삭제 → `VERTEX_PROJECT=metal-density-490000-m1`
- `VERTEX_LOCATION=global` (`.env`에 추가 — 아래 §2 참조)

## 2. Vertex AI 리전 문제 해결 — gemini-3.1-flash-image-preview 404

### 문제
Vertex AI ADC 전환 후 `gemini-3.1-flash-image-preview` 모델 호출 시 404 NOT_FOUND 오류 발생.
`us-central1`, `us-east4`, `us-west1`, `europe-west1` 모든 리전에서 동일.

### 근본 원인
`gemini-3.1-flash-image-preview`와 `gemini-3-pro-image-preview`는 Vertex AI에서 **`global` 리전 전용**.
기존 API Key 방식은 리전 지정이 불필요했으나, Vertex AI SDK는 `location` 파라미터가 필수.
코드 기본값이 `us-central1`이어서 404 발생.

### 해결
`.env`에 `VERTEX_LOCATION=global` 추가. 코드에서 `os.environ.get("VERTEX_LOCATION", "us-central1")`로 읽으므로 `.env` 1줄 추가로 해결.

### 검증
- `gemini-2.5-flash-image` (us-central1): OK — 이 모델은 리전 제한 없음
- `gemini-3.1-flash-image-preview` (global): OK
- `gemini-3-pro-image-preview` (global): 미확인 (동일 리전이므로 동작 예상)
- CH03 run003 shot00 (TITLECARD) 이미지 생성 성공

### 참고
- [Gemini 3.1 Flash Image 공식 문서](https://docs.cloud.google.com/vertex-ai/generative-ai/docs/models/gemini/3-1-flash-image) — 리전: Global

## 4. ref_images 파이프라인 정합성 근본 개선

### 문제

이미지 생성 시 반복되는 문제(캐릭터 누락, 잘못된 ref 참조)의 근본 원인:
1. generate_images.py가 11가지 자체 판단 로직 내장 → visual_direction 파일과 실제 API 전달 내용이 다름
2. costume_refs: [main] 컨벤션이 비직관적 → 96.9% 위반율
3. SR 규칙 12개가 너무 많아 에이전트가 실행하지 않음

### 해결: 3가지 구조 개선

**A. 태그 재설계 — costume_refs: [] = 기본 해빛 (정상)**
- `costume_refs: [main]` 컨벤션 폐기. 빈 배열 = 변장 없는 기본 해빛
- has_human이 ref_images 캐릭터 소스의 유일한 결정 키
- run003의 31건 위반이 즉시 정상화됨

**B. generate_images.py 역할 축소 — 단순 전달자**
- 판단 로직 9개 제거 + v2 코드 전체 제거 (1266줄 → 1009줄, -257줄)
- visual-director가 ref_images를 완전 구성 (style_ref + 캐릭터 ref + 소품 ref 모두 포함)
- generate_images.py는 ref_images를 그대로 API에 전달 (추가/제거 없음)

**C. SR 규칙 3그룹 재편 (12개 → 11개, 3그룹)**
- 그룹 A (필수, 3개): has_human↔creative_intent, narration_map 동일, NB2 금지어 — **위반 시 저장 불가**
- 그룹 B (품질, 4개): 시각 연속성, 연속 한도, 표정 묘사, 공간 배치 — 위반 시 경고
