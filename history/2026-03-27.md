## 2. Vertex AI 리전 문제 해결 — gemini-3.1-flash-image-preview 404

### 문제
Vertex AI ADC 전환 후 `gemini-3.1-flash-image-preview` 모델 호출 시 404 NOT_FOUND 오류 발생.
`us-central1`, `us-east4`, `us-west1`, `europe-west1` 모든 리전에서 동일.

### 근본 원인
`gemini-3.1-flash-image-preview`와 `gemini-3-pro-image-preview`는 Vertex AI에서 **`global` 리전 전용**.
기존 API Key 방식은 리전 지정이 불필요했으나, Vertex AI SDK는 `location` 파라미터가 필수.
코드 기본값이 `us-central1`이어서 404 발생.

### 해결
`.env`에 `VERTEX_LOCATION=global` 추가. 코드에서 `os.environ.get("VERTEX_LOCATION", "us-central1")`로 읽으므로 `.env` 1줄 추가로 해결.

### 검증
- `gemini-2.5-flash-image` (us-central1): OK — 이 모델은 리전 제한 없음
- `gemini-3.1-flash-image-preview` (global): OK
- `gemini-3-pro-image-preview` (global): 미확인 (동일 리전이므로 동작 예상)
- CH03 run003 shot00 (TITLECARD) 이미지 생성 성공

### 참고
- [Gemini 3.1 Flash Image 공식 문서](https://docs.cloud.google.com/vertex-ai/generative-ai/docs/models/gemini/3-1-flash-image) — 리전: Global

## 8. iv_prompt 개편 — Canvas vs Director 원칙

### 문제
shot-composer가 `[카메라] 모션: 슬로우 틸트업(3s) → 정지(2s)` 같은 상세한 카메라 안무와 `[감정선] 모션 아크: 시작→정점→여운`을 설계하지만, visual-director의 iv_prompt에서는 거의 "Static / 1% drift"로만 작성됨. 영상에 카메라 모션과 캐릭터 동작이 반영되지 않았음.

### 원인
iv-prompt-rules.md에 "최대 5~10% drift" 제한이 있었고, [CAMERA] 섹션이 1문장 수준으로 축소되어 있었음. shot-composer의 모션 시퀀스를 iv_prompt로 변환하는 매핑 규칙이 없었음.

### 해결: Canvas vs Director 대원칙 도입

**대원칙**: 이미지는 이미 Canvas(화면 구성)를 완성했다. iv_prompt는 Director(감독)로서 **동작 + 카메라 + 환경 변화**만 지시한다.

#### iv_prompt 구조 개편

| 기존 | 개편 |
|------|------|
| [ACTION] 1~2문장 | [CAMERA] — shot-composer 6종 모션 패턴 매핑 (1~2문장) |
| [CAMERA] 1문장, 5~10% drift | [ACTION] — 모션 아크 3단계(시작→정점→여운) 반영 (2~3문장) |
| — | [ENVIRONMENT] — 신규: 먼지/안개/조명변화/잉크 호흡 |
| [AUDIO] SFX+Ambient | [AUDIO] — 유지 |

#### [CAMERA] 6종 모션 패턴 변환 테이블

| shot-composer 패턴 | iv_prompt [CAMERA] |
|-------------------|-------------------|
| 하강 공개 (틸트다운) | `Slow tilt down, revealing character at bottom` |
| 상승 공개 (틸트업) | `Gentle tilt up from detail to full context` |
| 수평 추적 (패닝) | `Smooth pan left to right` |
| 줌 발견 (줌인) | `Slow push in toward the subject` |
| 여백 호흡 (줌아웃) | `Subtle pull back, expanding negative space` |
| 정지+미동 | `Nearly static, barely perceptible drift` |

#### [ACTION] Bad vs Good

```
❌ "The character stands still." (이미 이미지에 서 있으면 정지화면)
✅ "The character slowly lifts their head. At the apex, they freeze.
   Shoulders stiffen as they take one step back." (시작→정점→여운)
```

#### [ENVIRONMENT] emotion_tag별 기본값

| emotion_tag | 환경 효과 |
|-------------|----------|
| TENSION | 그림자 접근, 먼지 떨어짐, 미세 진동 |
| AWE | 빛 줄기 이동, 미세 입자 부유 |
| REVEAL | 조명 변화 (어두움→밝음) |
| HUMOR | 가벼운 바람, 종이 흔들림 |
| REFLECTIVE | 잉크 선 미세 호흡, 정적 속 미동 |

### 수정 파일 (5개)

| 파일 | 변경 |
|------|------|
| `visual-director/iv-prompt-rules.md` | **전면 개편** — Canvas vs Director, 4섹션 구조, 매핑 테이블, 통합 예시 3개 |
| `visual-director.md` | STEP 2.5 설명 + Self-Reflection에 "Static 남발 금지", "구체적 동사" 체크 추가 |
| `visual-director/self-check-per-shot.md` | F-0 섹션 신규 — iv_prompt 5항목 검증 체크리스트 |
| `CLAUDE.md` | iv_prompt 필드 설명 업데이트 |
| `pipeline_reference.md` | iv_prompt 비고 업데이트 |

### 검증 결과
- "5~10% drift" 제한 문구: 0건 (완전 제거)
- [ENVIRONMENT] 섹션: iv-prompt-rules.md + self-check에 반영 확인
- Canvas vs Director: 4개 파일에 반영 확인

## 9. 메인 캐릭터 채색 규칙 변경 — ref 시트 색 그대로

### 문제
has_human: main일 때 image_prompt P4에서 visual-director가 캐릭터 색을 직접 지정("넥커치프에 muted sage 워시")하고 있었으나, 턴어라운드 시트에 이미 캐릭터 고유 색이 정의되어 있어 중복/충돌 발생.

### 해결
스타일·얼굴을 ref 이미지에 위임한 것과 동일 원칙으로, **캐릭터 색도 ref 이미지(턴어라운드 시트)에 위임**.

| has_human | 캐릭터 색 | 장면 포인트 색 |
|-----------|----------|--------------|
| **main** | ref 시트 그대로 (P4 지정 안 함) | 1곳 필수 지정 |
| **anonym** | character_reference 그대로 | 1곳 필수 지정 |
| **none** | — | 1곳 필수 지정 |

#### P4 채색 문구 변경

```
기존: "반드시 {캐릭터}에 {색1} 워시를, {장면}에 {색2} 워시를 입혀줘 — 이 두 곳만이 유일한 색"
변경: "반드시 {장면 대상}에만 {색} 워시를 입혀줘 — 캐릭터는 참조 이미지의 색을 그대로 유지해줘"
```
