# 2026-03-20 실험 기록

## 실험 1: NB2 레퍼런스 이미지 참조 시 최적 프롬프트 설계

### 문제의식

현재 flow_prompt는 레퍼런스 이미지를 함께 전달하면서도 텍스트에서 캐릭터/소품의 외형을 **중복 묘사**하고 있다.

예 (shot12 현재 프롬프트):
```
첫 번째 이미지의 따뜻한 황토색 모자를 쓴 앞치마 캐릭터가 전체 화면의 약 10%,
거대한 블록의 5분의 1 크기로, 마치 산 앞에 선 개미처럼 뒤로 크게 젖히며...
캐릭터 얼굴은 코 없이, 작은 점 형태의 눈과 표현력 있는 벌어진 입으로만 그려줘.
첫 번째 이미지의 외형을 유지하되, scribbly한 손낙서 선으로 재해석해줘.
```

**의문**: ref_image로 캐릭터 이미지를 이미 전달하는데, "황토색 모자를 쓴 앞치마"라는 외형 묘사가 필요한가? "코 없이, 점 형태의 눈"도 이미지에 다 있는데?

### NB2 API 구조 이해

현재 API 호출 시 content 배열:
```
[
  "참조 이미지 (캐릭터 외형 기준 — ...같은 한 명이야):",   ← 라벨
  <artisan.jpeg 이미지 바이너리>,                         ← ref image
  "참조 이미지 (소품 — 형태와 디자인을 이 이미지 기준으로 재현):",
  <gear.jpeg 이미지 바이너리>,
  "<flow_prompt 전문>"                                   ← 텍스트 프롬프트
]
```

즉, NB2는 **라벨 + 이미지 + flow_prompt** 세 가지를 동시에 받는다.

### 실험 설계

**대조군**: 현재 flow_prompt (외형 묘사 풀 포함)
**실험군**: 외형 묘사를 단계적으로 줄여본다

| 변형 | 제거 대상 | 유지 대상 |
|------|----------|----------|
| A (현행) | - | 외형 묘사 + REDRAW 규칙 + 얼굴 규칙 + 크기/위치 + 감정/포즈 |
| B (외형 최소화) | 외형 묘사 ("황토색 모자를 쓴 앞치마") | 서수 참조 ("첫 번째 이미지의 캐릭터") + 크기/위치 + 감정/포즈 + REDRAW + 얼굴 |
| C (REDRAW도 제거) | 외형 묘사 + REDRAW | 서수 참조 + 크기/위치 + 감정/포즈 + 얼굴 |
| D (얼굴도 제거) | 외형 묘사 + REDRAW + 얼굴 규칙 | 서수 참조 + 크기/위치 + 감정/포즈만 |
| E (극한 최소) | 외형+REDRAW+얼굴+스타일 | 서수 참조 + 크기/위치 + 감정/포즈 + 장면 가드만 |

**테스트 Shot**: shot12 (AWE, 캐릭터+소품, 단순 구성 — 변수 적음)

### 평가 기준

1. 캐릭터 외형 일관성 (ref_image와 일치하는가)
2. 구도/크기 정확도 (지시한 대로 배치되는가)
3. 스타일 일관성 (Sempé ink 스타일 유지되는가)
4. 불필요한 요소 발생 여부 (텍스트, 다중 패널 등)

### 실험 1 결론

**ref_image가 담당 → 텍스트에서 제거 가능:**
| 항목 | 현행 프롬프트 | 제거 가능 여부 | 근거 |
|------|-------------|--------------|------|
| 캐릭터 외형 묘사 | "황토색 모자를 쓴 앞치마" | ✅ 제거 가능 | B에서 확인 — ref_image만으로 재현 |
| REDRAW 규칙 | "외형을 유지하되 scribbly한 선으로 재해석" | ✅ 제거 가능 | C에서 확인 — P2 스타일 + API 라벨이 역할 대체 |
| 얼굴 규칙 | "코 없이, 점 형태의 눈" | ✅ 제거 가능 | D에서 확인 — ref_image 얼굴 디자인을 따름 |

**텍스트만 담당 → 제거 불가:**
| 항목 | 이유 |
|------|------|
| 스타일 (P2) | ref_image 라벨이 "외형/형태 참조"이지 "스타일 참조"가 아님 |
| 배경 | ref_image에 배경 정보 없음 |
| 크기/위치/구도 | 장면 구성은 텍스트에서만 지시 |
| 감정/포즈 | 동적 상태는 텍스트에서만 지시 |
| 색 계획 (P5) | 채색 지시는 텍스트 전용 |
| 장면 가드 | "하나의 장면만" 등 제어 장치 |

**프롬프트 절감 효과**: 캐릭터당 ~80자 절약 (외형+REDRAW+얼굴). 2캐릭터 shot이면 ~160자.

**파이프라인 반영 방향**: visual-director의 P3에서 외형 묘사·REDRAW·얼굴 규칙을 제거하고, 서수 참조("첫 번째 이미지의 캐릭터")만 유지.

## 실험 2: 스타일 참조를 ref_image로 이전할 수 있는가

### 문제의식

실험 1에서 "스타일은 텍스트에서 제거 불가"로 결론지었으나, 만약 **스타일 참조용 이미지**를 추가로 전달하거나 **API 라벨을 수정**하여 "이 이미지의 스타일도 따라라"고 지시하면 P2 스타일 묘사도 줄일 수 있지 않을까?

### 현재 구조

```
API content 배열:
  "참조 이미지 (캐릭터 외형 기준 — ...체형·의상·비율을 따라...):"  ← 외형만 참조
  <artisan.jpeg>
  "참조 이미지 (소품 — 형태와 디자인을 이 이미지 기준으로 재현):"   ← 형태만 참조
  <gear.jpeg>
  "<flow_prompt>"                                                  ← 스타일은 여기서만
```

### 실험 설계

| 변형 | 방법 | 질문 |
|------|------|------|
| E1 | API 라벨에 "스타일도 따라 그려줘" 추가 | 라벨 변경만으로 스타일 전이 가능한가? |
| E2 | 스타일 전용 ref_image 추가 (기존 생성 이미지 1장을 스타일 샘플로) | 별도 스타일 이미지가 효과적인가? |
| E3 | E1 또는 E2 성공 시, P2 스타일 묘사를 최소화 | 어디까지 줄일 수 있는가? |

### 평가 기준
1. 잉크 선 품질 (가느다란 떨림, 해칭)
2. 워터컬러 워시 (번짐, 투명감)
3. 분위기 (whimsical, heartwarming)
4. 배경 순백 유지

### 실험 2 결론

**NB2의 스타일 결정 메커니즘:**

```
캐릭터/소품 ref_image의 드로잉 스타일 >>>>> 텍스트 스타일 묘사 ≈ style_ref 이미지 ≈ 라벨 지시
```

- **style_ref 이미지**: 라벨 강화 + 프롬프트 명시 참조까지 해도 효과 없음 (G, I, J, K에서 확인)
- **텍스트 스타일 묘사 (P2)**: 있어도 없어도 결과 동일 (E3b vs F에서 확인)
- **ref_image 자체의 스타일**: 출력을 완전히 지배. Phase 1에서 생성한 이미지 스타일 = 최종 출력 스타일.

**발견 — 라벨 네이밍 (THIS {name}):**
- NB2는 라벨에 부여한 이름을 프롬프트에서 참조할 수 있음 (H에서 확인)
- "THIS artisan"으로 캐릭터를 직접 참조하면 서수("첫 번째 이미지의")와 동일하게 작동
- 복수 캐릭터 shot에서 명확성 향상 가능 (추가 검증 필요)

**실질적 의미:**
- P2 스타일 묘사는 **제거해도 무방** — ref_image가 이미 스타일을 결정
- style_ref는 **불필요** — ref_image의 스타일 영향력을 이길 수 없음
- 스타일을 바꾸려면 **Phase 1 ref_image 자체를 다른 스타일로 생성**해야 함
- 라벨 네이밍은 **유의미** — 서수 대신 이름 참조가 작동하며, 프롬프트 가독성 향상

## 실험 1+2 종합: 파이프라인 반영 방향

### 수정 대상 에이전트/스킬

**1. visual-director (STEP 06) — flow_prompt 생성 규칙 변경**

현행 5단락 중 제거 가능한 부분:

| 단락 | 현행 | 변경 | 근거 |
|------|------|------|------|
| P1 | 과제 소개 | 유지 | 장면 가드 역할 |
| P2 | 스타일 묘사 + 배경 | **스타일 묘사 제거**, 배경만 유지 | 실험 2-F: ref_image가 스타일 결정 |

## 실험 7: 전체 캐릭터(4) + 소품(9) 복합 씬 — THIS 네이밍 동시 제어

### 문제의식
