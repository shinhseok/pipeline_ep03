# M — Gemini API 최종 전달 content

## 목적
ref_image가 없는 shot(has_human: false, ref_images: [])에서
style3_ref.png(Sempé 완성 씬)를 스타일 참조로 전달하면 스타일이 통제되는지 검증.

실험 1+2에서 "ref_image가 스타일을 지배"를 확인했으나,
ref_image가 없는 shot에서는 스타일 전이 소스가 없어 스타일이 제각각이었음.
이 경우 style_ref가 유일한 스타일 소스가 되므로 효과가 있을 수 있음.

## API에 전달된 content 배열 (4개 shot 공통 구조)

```
THIS style — 이 이미지의 드로잉 스타일로 장면 전체를 그려줘:
```
`<style3_ref.png>` (Sempé 스타일 완성 씬 — 마을 풍경, 잉크선 + 워시)
```
{shot별 flow_prompt — 스타일 묘사 제거, "제공된 참조 이미지의 드로잉 스타일로" 시작}
```

## Shot별 flow_prompt

### shot14 (끊어진 실)
```
제공된 참조 이미지의 드로잉 스타일로, 유튜브 교육 영상에 사용할
서사적인 삽화 한 장을 그려줘.

배경은 아무런 색이나 질감이 없는 순백의 빈 공간으로 남겨둬.

화면 중앙에 물레에서 나온 실 한 올이 수직으로 늘어져 있어 —
전체 화면의 약 10%, 중간에서 가늘어지다 끊어진 채 끝이 아래로
힘없이 축 늘어져 있어. 끊어진 실의 아래쪽 끝이 바닥을 향해
힘없이 떨어지는 순간이야. 나머지 화면은 순백의 여백이야.

반드시 실에만 faded blue 워시를 입혀줘 — 이것이 이 그림의 유일한 색이야.
하나의 장면만 그려줘. 정확히 끊어진 실 한 올만 그려줘.
```

## 현행(A) 대비 변경점
- P2 스타일 묘사 ("Scribbly, hand-drawn doodle...") 제거
- P5 "흑백의 잉크 선으로만" 제거
- P1 "제공된 참조 이미지의 드로잉 스타일로" — style_ref 명시 참조
- style3_ref.png를 content 맨 앞에 THIS style 라벨로 전달

## 설정
- model: gemini-3.1-flash-image-preview
- thinking_level: HIGH

## 프롬프트 수정 필요 사항 (다음 실험에서 반영)
- "제공된 참조 이미지의 드로잉 스타일로" → "THIS style의 드로잉 스타일로"
- 라벨 네이밍 전략과 일관성을 맞추려면 프롬프트에서도 THIS style을 직접 참조해야 함
