# G — Gemini API 최종 전달 content

## 목적
style_ref 라벨을 강화하고 캐릭터/소품 라벨에서 스타일을 분리해서,
style_ref(굵은 해칭, 강한 대비)의 스타일이 실제로 적용되는지 검증.

## API에 전달된 content 배열 (순서대로)

```
최우선 스타일 참조 — 반드시 이 이미지의 드로잉 스타일(선 굵기, 해칭 밀도, 음영 방식, 채색 기법)을 장면 전체에 적용해. 아래 다른 참조 이미지에서는 외형만 가져오고, 드로잉 스타일은 이 이미지를 따라:
```
`<style_ref.png>`
```
참조 이미지 — 이 이미지에서 외형(체형, 의상, 비율)만 가져와. 드로잉 스타일은 위의 스타일 참조를 따라:
```
`<artisan.jpeg>`
```
참조 이미지 — 이 이미지에서 형태와 디자인만 가져와. 드로잉 스타일은 위의 스타일 참조를 따라:
```
`<gear.jpeg>`
```
제공된 참조 이미지들의 외형을 바탕으로, 유튜브 교육 영상에 사용할
서사적인 삽화 한 장을 그려줘.

배경은 순백의 빈 공간이되, 바닥선이 희미하게 암시되어 있어.

먼저 화면 좌측 상단을 보면 — 거대한 실타래 블록이 캐릭터 키의
5배 높이로 솟아 있어. 꼭대기가 프레임 밖으로 잘릴 정도야. 실타래를
쌓아올린 듯한 추상적 블록 형태야. 그 옆 우측 하단에 작은 블록이
캐릭터 키의 0.3배 크기로 놓여 있어. 두 블록 사이 아래, 화면 하단
중앙에 첫 번째 이미지의 캐릭터가
전체 화면의 약 10%, 거대한 블록의 5분의 1 크기로, 마치 산 앞에
선 개미처럼 뒤로 크게 젖히며 팔을 쭈욱 뻗어 올린 채 올려다보고
있어. 고개가 다 올라갔는데도 블록의 꼭대기가 보이지 않는 바로 그
찰나에 멈춰 있어. 눈이 동그랗게 커지고 입이 살짝 벌어진 표정이야.

반드시 거대 블록에만 deep amber 워시를 입혀줘 — 이것이 이 그림의 유일한 색이야.
하나의 장면만 그려줘. 정확히 캐릭터 1명과 블록 2개만 그려줘.
```

## 설정
- model: gemini-3.1-flash-image-preview
- thinking_level: HIGH

## 결과
style_ref의 "굵은 해칭, 강한 대비" 스타일이 적용되지 않음.
여전히 artisan.jpeg/gear.jpeg의 부드러운 Sempé 스타일이 우세.

## 핵심 발견
NB2는 라벨 텍스트로 "스타일은 이 이미지를 따라"라고 지시해도,
캐릭터/소품 ref_image 자체의 드로잉 스타일이 출력을 지배함.
