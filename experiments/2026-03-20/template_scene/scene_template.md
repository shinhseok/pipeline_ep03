# 씬 이미지 생성 템플릿 (NB2 — 실험 1~8 확정)

## API content 배열 구조

```
THIS style — 이 이미지의 드로잉 스타일로 장면 전체를 그려줘:
<style_ref.png — 캐릭터 없는 앵커 이미지, 항상 첨부>

THIS {캐릭터명} — 이 대상의 형태만 따라 그려줘:
<캐릭터 턴어라운드 시트>

THIS {소품명} — 이 대상의 형태만 따라 그려줘:
<소품 턴어라운드 시트>

(... 필요한 만큼 반복 ...)

<flow_prompt>
```

## flow_prompt 템플릿

```
THIS style의 드로잉 스타일로, 유튜브 교육 영상에 사용할
서사적인 삽화 한 장을 그려줘.

{배경 지시}

{구도/배치/크기 — THIS {name}으로 직접 참조}
{감정/포즈}

{채색 포인트 — "반드시 X에만 Y 워시를 입혀줘 — 유일한 색이야."}
```

## 규칙

| 항목 | 규칙 |
|------|------|
| 라벨 | "THIS {name} — 이 대상의 형태만 따라 그려줘:" (캐릭터/소품 동일) |
| style_ref | 항상 첨부. 캐릭터가 없는 이미지만 사용 |
| ref 형식 | 캐릭터/소품 모두 턴어라운드 시트 |
| 이름 참조 | 프롬프트에서 THIS {name}으로 직접 참조 (서수 불사용) |
| 외형 묘사 | 불필요 — ref 이미지가 담당 |
| 스타일 묘사 | 불필요 — ref 이미지 스타일이 지배 |
| REDRAW | 불필요 |
| 얼굴 규칙 | 불필요 — ref 이미지가 담당 |
| 카운트 제약 | 불필요 — 구도 묘사에서 요소를 명시 |
| 장면 가드 | 불필요 — 구도 묘사에서 요소를 명시 |

## 프롬프트에 남는 것만

1. **과제 소개**: "THIS style의 드로잉 스타일로, ... 삽화 한 장을 그려줘."
2. **배경**: "배경은 순백의 빈 공간이되, ..."
3. **구도/배치/크기**: THIS {name} + 위치 + 크기 비율
4. **감정/포즈**: 표정, 자세, 결정적 순간
5. **채색 포인트**: 워시 색상과 위치

## 예시 (shot12 최적화 버전)

```
THIS style의 드로잉 스타일로, 유튜브 교육 영상에 사용할
서사적인 삽화 한 장을 그려줘.

배경은 순백의 빈 공간이되, 바닥선이 희미하게 암시되어 있어.

먼저 화면 좌측 상단을 보면 — 거대한 실타래 블록이 THIS main 키의
5배 높이로 솟아 있어. 꼭대기가 프레임 밖으로 잘릴 정도야. 실타래를
쌓아올린 듯한 추상적 블록 형태야. 그 옆 우측 하단에 작은 블록이
THIS main 키의 0.3배 크기로 놓여 있어. 두 블록 사이 아래, 화면 하단
중앙에 THIS main이 전체 화면의 약 10%, 거대한 블록의 5분의 1 크기로,
마치 산 앞에 선 개미처럼 뒤로 크게 젖히며 팔을 쭈욱 뻗어 올린 채
올려다보고 있어. 고개가 다 올라갔는데도 블록의 꼭대기가 보이지 않는
바로 그 찰나에 멈춰 있어. 눈이 동그랗게 커지고 입이 살짝 벌어진
표정이야.

반드시 거대 블록에만 deep amber 워시를 입혀줘 — 유일한 색이야.
```
