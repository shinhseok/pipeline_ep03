# R — 카운트 제약 필요성 검증

## 목적
프롬프트 마지막 줄의 카운트 제약을 단계적으로 제거했을 때 결과 차이 검증.

## 확정 템플릿 적용 (1~4번 확정사항)
- 라벨: "THIS {name} — 이 대상의 형태만 따라 그려줘:" (통일)
- style_ref: 항상 첨부 (ref_image 유무 무관)
- 소품: 모두 턴어라운드 시트 (단일 뷰 구분 없음)

## 3가지 변형

### R1 — 카운트 + 장면 가드 (현행)
```
...
하나의 장면만 그려줘. 정확히 THIS main 1명과 블록 2개만 그려줘.
```

### R2 — 장면 가드만 (카운트 제거)
```
...
하나의 장면만 그려줘.
```

### R3 — 둘 다 제거
```
(마지막 줄 없음)
```

## 공통 API content 구조

```
THIS style — 이 이미지의 드로잉 스타일로 장면 전체를 그려줘:
```
`<assets/reference/anchor/style_ref.png>`
```
THIS main — 이 대상의 형태만 따라 그려줘:
```
`<O_multiview_ref/O_sheet_v2.png>`
```
{flow_prompt — 마지막 줄만 다름}
```

## 설정
- model: gemini-3.1-flash-image-preview
- thinking_level: HIGH

## 평가 기준
1. 캐릭터 수 — 1명만 나오는가 (추가 캐릭터 발생 여부)
2. 블록 수 — 2개만 나오는가
3. 다중 패널 발생 — 한 장면이 아닌 여러 장면이 생기는가
4. 전체 품질 차이

## 결과

| 기준 | R1 (카운트+가드) | R2 (가드만) | R3 (둘 다 없음) |
|------|----------------|-----------|---------------|
| 캐릭터 수 | 1명 ✅ | 1명 ✅ | 1명 ✅ |
| 블록 수 | 2개 ✅ | 2개 ✅ | 2개 ✅ |
| 다중 패널 | 없음 ✅ | 없음 ✅ | 없음 ✅ |
| 품질 차이 | 없음 | 없음 | 없음 |

**결론: 카운트 제약 + 장면 가드 모두 제거 가능.**
프롬프트 본문에서 "THIS main이 화면 하단 중앙에"와 같이 요소를 명시하면
별도의 카운트/가드 문장이 불필요.
